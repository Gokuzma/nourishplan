### 8. FEED-02 — dietary restrictions [BUG FOUND]

- Navigated to Settings → ticked a dietary restriction checkbox (index 6 = Halal).
- Clicked "Save restrictions".
- **Network:** `POST dietary_restrictions?on_conflict=household_id,member_user_id { predefined:["Halal"], ... }` → **400 Bad Request**.
- **Root cause (code review `src/hooks/useDietaryRestrictions.ts:55`):** `.upsert(row, { onConflict: 'household_id,member_user_id' })` — but `supabase/migrations/024_feedback_dietary.sql:64-66` defines the uniqueness as a **function-based partial index** `create unique index dietary_restrictions_member_unique on dietary_restrictions(coalesce(member_user_id::text, member_profile_id::text))`. PostgreSQL rejects the upsert because no `UNIQUE(household_id, member_user_id)` constraint matches the `ON CONFLICT` clause.
- **Impact:** FEED-02 save flow is broken. Users cannot save dietary restrictions.
- **Suggested fix:** Add `alter table dietary_restrictions add constraint dietary_restrictions_user_unique unique (household_id, member_user_id)` (with partial NULLS FIRST handling) + symmetric one for `member_profile_id`; OR rewrite the hook to use query-then-insert-or-update instead of upsert.
- **Status: REMAINS Partial** (documented bug — does not force `gaps_found` but flagged for future fix)

## New bugs found

### BUG-UAT-01: FEED-02 dietary restrictions save returns 400

- **File:** `src/hooks/useDietaryRestrictions.ts:55`
- **Symptom:** Clicking "Save restrictions" on Settings page returns `400 Bad Request` from PostgREST.
- **Root cause:** Upsert uses `onConflict: 'household_id,member_user_id'` but `supabase/migrations/024_feedback_dietary.sql:64-66` defines uniqueness via a function-based partial index `dietary_restrictions_member_unique ON (coalesce(member_user_id::text, member_profile_id::text))` — no `UNIQUE (household_id, member_user_id)` constraint exists.
- **Fix (one-line direction):** Either (a) add a tuple unique constraint via new migration — `alter table dietary_restrictions add constraint dietary_restrictions_user_unique unique (household_id, member_user_id) deferrable initially deferred` + symmetric `member_profile_id`; OR (b) rewrite the hook to query-then-insert-or-update instead of PostgREST upsert.
- **Severity:** Medium — blocks FEED-02 save flow. FEED-03 (won't-eat) works because it uses a different table and insert pattern.
- **Filed for:** Future milestone or gap-closure phase.
