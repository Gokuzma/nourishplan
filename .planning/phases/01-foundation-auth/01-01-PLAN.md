<task type="auto">
  <name>Task 2: Create database migration with full Phase 1 schema and RLS policies</name>
  <files>supabase/migrations/001_foundation.sql</files>
  <action>
Create `supabase/migrations/001_foundation.sql` containing the complete Phase 1 schema. Use the exact SQL from the research document (01-RESEARCH.md) as the foundation, with these tables and policies:

1. **profiles** table:
   - `id uuid primary key references auth.users(id) on delete cascade`
   - `display_name text`, `avatar_url text`, `created_at timestamptz default now()`
   - RLS: users see/update own profile only
   - Trigger `handle_new_user()` on `auth.users` insert — auto-creates profile row. Wrap in exception handler so signup never fails due to trigger error.

2. **households** table:
   - `id uuid primary key default gen_random_uuid()`
   - `name text not null`, `created_at timestamptz default now()`
   - RLS: members see only their own household

3. **household_role** enum: `('admin', 'member')`

4. **household_members** table:
   - `id uuid primary key default gen_random_uuid()`
   - `household_id uuid references households(id) on delete cascade`
   - `user_id uuid references auth.users(id) on delete cascade`
   - `role household_role default 'member'`
   - `joined_at timestamptz default now()`
   - `unique (household_id, user_id)`
   - RLS policies:
     - Members can read their own membership row
     - Members can read all members of their own household (for member list)
     - Admins can insert new members (for invite acceptance)
     - Authenticated users can insert their own membership row (for household creation — creator adds self as admin)

5. **household_invites** table:
   - `id uuid primary key default gen_random_uuid()`
   - `household_id uuid references households(id) on delete cascade`
   - `token text not null unique default encode(gen_random_bytes(16), 'hex')`
   - `created_by uuid references auth.users(id)`
   - `expires_at timestamptz default now() + interval '7 days'`
   - `used_at timestamptz`, `created_at timestamptz default now()`
   - RLS: admins can create invites; anyone authenticated can read valid (non-expired, unused) invites by token (for join flow)

6. **member_profiles** table:
   - `id uuid primary key default gen_random_uuid()`
   - `household_id uuid references households(id) on delete cascade`
   - `managed_by uuid references auth.users(id)`
   - `name text not null`, `is_child boolean default false`, `birth_year int`
   - `created_at timestamptz default now()`
   - RLS: managed_by user can CRUD their own managed profiles

7. Enable RLS on ALL tables. Use `(select auth.uid())` pattern (not bare `auth.uid()`) for performance per Supabase recommendation.

8. Add INSERT policy on `households` for authenticated users (anyone can create a household).

CRITICAL: Ensure the `household_members` INSERT policy allows a user to insert their own row when creating a household (bootstrap problem — creator has no existing membership yet). Add a policy: authenticated users can insert a row where `user_id = (select auth.uid())` AND `role = 'admin'` AND no existing membership exists for that user (one household per user constraint).

This file is a reference migration — user applies it via Supabase Dashboard SQL Editor or `supabase db push`.
