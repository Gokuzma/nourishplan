- carbs_per_100g (numeric NOT NULL)
- fiber_per_100g (numeric)

- deleted_at (timestamptz) — soft delete
- created_at (timestamptz NOT NULL DEFAULT now())
- updated_at (timestamptz NOT NULL DEFAULT now())

**recipes:**
- id (uuid PK default gen_random_uuid())
- household_id (uuid NOT NULL references households(id))
- created_by (uuid NOT NULL references auth.users(id))
- name (text NOT NULL)
- servings (integer NOT NULL DEFAULT 1 CHECK (servings > 0))
- deleted_at (timestamptz) — soft delete
- created_at (timestamptz NOT NULL DEFAULT now())
- updated_at (timestamptz NOT NULL DEFAULT now())

**recipe_ingredients:**
- id (uuid PK default gen_random_uuid())
- recipe_id (uuid NOT NULL references recipes(id) ON DELETE CASCADE)
- ingredient_type (text NOT NULL CHECK (ingredient_type IN ('food', 'recipe')))
- ingredient_id (uuid NOT NULL) — refs custom_foods.id OR recipes.id depending on type
- quantity_grams (numeric NOT NULL)
- weight_state (text NOT NULL DEFAULT 'raw' CHECK (weight_state IN ('raw', 'cooked')))
- sort_order (integer NOT NULL DEFAULT 0)
- created_at (timestamptz NOT NULL DEFAULT now())

Enable RLS on all three tables. Add these policies:

**custom_foods:**
- SELECT: household members can read (same pattern as existing household RLS — check household_members join)
- INSERT: authenticated users can insert into their household (household_id must match their membership)
- UPDATE/DELETE: creator OR household admin only
- All SELECT/UPDATE/DELETE policies filter out deleted_at IS NOT NULL rows for SELECT (soft delete transparency)

**recipes:** Same RLS pattern as custom_foods (household-scoped read, creator/admin modify, soft delete filter on SELECT)

**recipe_ingredients:** SELECT/INSERT/UPDATE/DELETE allowed if user has access to the parent recipe (join through recipes table to household_members)

Add updated_at trigger function (reuse if exists from 001, else create) on custom_foods and recipes.
