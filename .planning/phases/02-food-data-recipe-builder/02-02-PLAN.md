<task type="auto">
  <name>Task 1: USDA and Open Food Facts Edge Functions</name>
  <files>supabase/functions/search-usda/index.ts, supabase/functions/search-off/index.ts</files>
  <action>
Create `supabase/functions/` directory structure. Each Edge Function is a directory with an `index.ts`.

**search-usda/index.ts:**
- Import serve from Deno std http
- Accept POST with JSON body: `{ query: string, pageSize?: number }`
- Read `USDA_API_KEY` from `Deno.env.get('USDA_API_KEY')`
- Fetch `https://api.nal.usda.gov/fdc/v1/foods/search` with params: api_key, query (URI-encoded), dataType=Foundation,SR%20Legacy,Survey%20(FNDDS),Branded, pageSize (default 25)
- Deduplicate results by food description: prioritize Foundation > SR Legacy > Survey > Branded. For each unique description (case-insensitive), keep only the highest-priority dataType.
- Normalize each result to: `{ fdcId, name: description, source: 'usda', dataType, calories, protein, fat, carbs, portions }`. Extract nutrients by number: 208=calories, 203=protein, 204=fat, 205=carbs. Default to 0 if not found. Extract portions from `foodMeasures` array as `{ description: string, grams: number }`.
- Return JSON array with CORS headers: `Access-Control-Allow-Origin: *`, `Access-Control-Allow-Headers: authorization, x-client-info, apikey, content-type`
- Handle OPTIONS preflight request
- Handle errors: if USDA API fails, return `{ error: "USDA search unavailable" }` with status 502
- If USDA_API_KEY is not set, return `{ error: "USDA API key not configured" }` with status 500

**search-off/index.ts:**
- Accept POST with JSON body: `{ query: string, pageSize?: number }`
- Fetch `https://world.openfoodfacts.org/cgi/search.pl` with params: search_terms (URI-encoded), search_simple=1, action=process, json=1, page_size (default 25), fields=product_name,nutriments,serving_size,brands,code
- Add User-Agent header: `NourishPlan/1.0 (contact@nourishplan.app)` (required by OFF ToS)
- Filter out products where `energy-kcal_100g`, `proteins_100g`, `fat_100g`, AND `carbohydrates_100g` are ALL missing/null. Keep products that have at least calories present.
- Normalize each result to: `{ id: code, name: product_name + (brands ? " - " + brands : ""), source: 'off', calories: nutriments['energy-kcal_100g'] ?? 0, protein: nutriments['proteins_100g'] ?? 0, fat: nutriments['fat_100g'] ?? 0, carbs: nutriments['carbohydrates_100g'] ?? 0 }`
- Same CORS headers and error handling pattern as USDA function
- No API key needed for OFF (public API)

<task type="auto">
  <name>Task 2: AI nutrition verification Edge Function</name>
  <files>supabase/functions/verify-nutrition/index.ts</files>
  <action>
Create `supabase/functions/verify-nutrition/index.ts`:

- Accept POST with JSON body: `{ foodName: string, usdaValues?: MacroValues, offValues?: MacroValues }` where MacroValues is `{ calories, protein, fat, carbs }`
- Read `ANTHROPIC_API_KEY` from `Deno.env.get('ANTHROPIC_API_KEY')`
- If key not set, return `{ verified: false, reason: "AI verification not configured" }` with status 200 (graceful degradation — do not block food search)
- Call Claude Haiku (claude-haiku-4-5) via Anthropic messages API:
  - URL: `https://api.anthropic.com/v1/messages`
  - Headers: x-api-key, anthropic-version: 2023-06-01, content-type: application/json
  - Body: model claude-haiku-4-5, max_tokens 256
  - Prompt: Check nutrition values per 100g for "{foodName}". Include whichever sources are provided (USDA values, OFF values, or both). Ask for JSON response: `{ "verified": true/false, "recommended": { calories, protein, fat, carbs }, "reason": "...", "warnings": ["..."] }`
  - Parse the JSON from Claude's text response
- Outlier detection rules (run even with single source):
  - If protein + fat + carbs > calories/4 * 1.1 (macro-calorie math check: protein*4 + fat*9 + carbs*4 should approximate calories)
  - If any macro is negative or calories is 0 but macros sum > 0
  - Add warning strings for each detected issue
- Return: `{ verified: boolean, recommended: MacroValues, reason: string, warnings: string[] }`
- Same CORS headers pattern
- If Anthropic API call fails, return `{ verified: false, reason: "AI verification unavailable", warnings: [] }` with status 200 (graceful degradation)
