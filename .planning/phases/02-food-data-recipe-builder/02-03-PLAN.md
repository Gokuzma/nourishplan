**src/hooks/useFoodSearch.ts:**
- Export `useFoodSearch(tab: 'usda' | 'off', query: string)` using `useQuery` from TanStack Query
- queryKey: `['food-search', tab, query]`
- queryFn: if query is empty or < 2 chars, return []. Otherwise call `supabase.functions.invoke(tab === 'usda' ? 'search-usda' : 'search-off', { body: { query } })`. Cast result to `NormalizedFoodResult[]`.
- enabled: `query.length >= 2`
- staleTime: 5 * 60 * 1000 (cache 5 min per research recommendation)
- Return the useQuery result
- Handle error case: if invoke returns error, throw it

**src/hooks/useCustomFoods.ts:**
Follow the exact same patterns from `useHousehold.ts` (useQuery + useMutation with query invalidation).

- `useCustomFoods()`: fetches all custom_foods for the user's household (non-deleted). Query from supabase: `.from('custom_foods').select('*').eq('household_id', householdId).is('deleted_at', null).order('name')`. queryKey: `['custom-foods', householdId]`. enabled: `!!householdId`.

- `useCreateCustomFood()`: useMutation that inserts into custom_foods with household_id and created_by from session. Input shape: `{ name, serving_description, serving_grams, calories_per_100g, protein_per_100g, fat_per_100g, carbs_per_100g, fiber_per_100g?, sugar_per_100g?, sodium_per_100g?, micronutrients? }`. Invalidates `['custom-foods']` on success.

- `useUpdateCustomFood()`: useMutation accepting `{ id, updates }`. Updates the row. Invalidates `['custom-foods']`.

- `useDeleteCustomFood()`: useMutation that soft-deletes by setting `deleted_at` to now(). Invalidates `['custom-foods']`.

All hooks import useAuth, useHousehold for session/household context, matching the existing pattern.
