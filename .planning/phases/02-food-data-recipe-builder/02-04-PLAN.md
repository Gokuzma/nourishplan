  <action>
Create `src/hooks/useRecipes.ts` following the TanStack Query patterns from useHousehold.ts.

**Query hooks:**
- `useRecipes()`: fetches all recipes for user's household (non-deleted). `.from('recipes').select('*').eq('household_id', householdId).is('deleted_at', null).order('name')`. queryKey: `['recipes', householdId]`.

- `useRecipe(id: string)`: fetches single recipe. `.from('recipes').select('*').eq('id', id).single()`. queryKey: `['recipe', id]`. enabled: `!!id`.

- `useRecipeIngredients(recipeId: string)`: fetches ingredients for a recipe. `.from('recipe_ingredients').select('*').eq('recipe_id', recipeId).order('sort_order')`. queryKey: `['recipe-ingredients', recipeId]`. enabled: `!!recipeId`.

**Mutation hooks:**
- `useCreateRecipe()`: inserts recipe with household_id and created_by. Input: `{ name, servings }`. Returns created recipe. Invalidates `['recipes']`.

- `useUpdateRecipe()`: updates recipe name or servings. Input: `{ id, updates: { name?, servings? } }`. Invalidates `['recipes']` and `['recipe', id]`.

- `useDeleteRecipe()`: soft deletes recipe (sets deleted_at). Invalidates `['recipes']`.

- `useAddIngredient()`: inserts into recipe_ingredients. Input: `{ recipe_id, ingredient_type, ingredient_id, quantity_grams, weight_state?, sort_order? }`. Invalidates `['recipe-ingredients', recipe_id]`.

- `useUpdateIngredient()`: updates recipe_ingredients row. Input: `{ id, recipe_id, updates: { quantity_grams?, weight_state?, sort_order? } }`. Invalidates `['recipe-ingredients', recipe_id]`.

- `useRemoveIngredient()`: deletes recipe_ingredients row. Input: `{ id, recipe_id }`. Invalidates `['recipe-ingredients', recipe_id]`.

All hooks use useAuth and useHousehold for context, matching existing patterns.
