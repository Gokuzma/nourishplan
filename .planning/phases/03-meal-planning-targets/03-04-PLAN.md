  <action>
    Create `src/hooks/useMealPlan.ts` following TanStack Query patterns:

    **useMealPlan(weekStart):** Query meal_plan for current household and weekStart date. Use `.maybeSingle()` — returns null if no plan exists yet for this week. Query key: `['meal-plan', householdId, weekStart]`.

    **useCreateMealPlan():** Mutation to upsert a meal_plan row. Use `.upsert({ household_id, week_start, created_by }, { onConflict: 'household_id,week_start', ignoreDuplicates: true })` then `.select().single()` to handle the race condition (pitfall 1 in RESEARCH.md). Invalidate `['meal-plan', householdId, weekStart]`.

    **useMealPlanSlots(planId):** Query all slots for a plan, joined with meals and their meal_items: `.select('*, meals(*, meal_items(*))').eq('plan_id', planId).order('day_index').order('slot_order')`. Query key: `['meal-plan-slots', planId]`. Enabled when planId truthy.

    **useAssignSlot():** Mutation to upsert a slot row. Accept `{ planId, dayIndex, slotName, slotOrder, mealId, isOverride }`. Upsert on `(plan_id, day_index, slot_name)`. Invalidate `['meal-plan-slots', planId]`.

    **useClearSlot():** Mutation to set meal_id = null on a slot. Invalidate `['meal-plan-slots', planId]`.

    The slot join with meals and meal_items allows computing nutrition client-side using calcIngredientNutrition + calcMealNutrition from existing utils — no server-side computation needed.
