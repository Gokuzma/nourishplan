  <action>
    **useMealPlanTemplates.ts:** Create hooks following TanStack Query patterns:

    **useTemplates():** Query all templates for household. Query key: `['meal-plan-templates', householdId]`.

    **useSaveAsTemplate():** Mutation that:
    1. Inserts a `meal_plan_templates` row with name and household_id
    2. Fetches current plan's slots via `meal_plan_slots` where plan_id matches
    3. For each slot, inserts a `meal_plan_template_slots` row with template_id, day_index, slot_name, slot_order, meal_id
    Templates reference meal IDs (not copies of meal items — per RESEARCH.md anti-pattern guidance).
    Invalidate `['meal-plan-templates', householdId]`.

    **useLoadTemplate():** Mutation that:
    1. Accepts `{ templateId, planId }` — the plan must already exist
    2. Fetches template slots from `meal_plan_template_slots` where template_id matches
    3. For each template slot, upserts into `meal_plan_slots` on `(plan_id, day_index, slot_name)` with the template's meal_id and `is_override = false`
    Invalidate `['meal-plan-slots', planId]`.

    **useDeleteTemplate():** Mutation to delete a template (cascade deletes its slots). Invalidate `['meal-plan-templates', householdId]`.

    **useRepeatLastWeek():** Mutation that:
    1. Accepts `{ currentPlanId, previousWeekStart }`
    2. Finds the previous week's plan by household_id + previousWeekStart
    3. Fetches its slots, copies them into current plan's slots via upsert
    Invalidate `['meal-plan-slots', currentPlanId]`.

    **NewWeekPrompt.tsx:** Modal component shown when PlanPage detects no meal_plan exists for the current weekStart (pitfall 4 — presence of a plan row means prompt was handled).

    Props: `{ weekStart: string, onChoice: (choice: 'fresh' | 'repeat' | 'template', templateId?: string) => void }`.

    Display three options:
    1. "Start fresh" — creates empty plan (calls useCreateMealPlan)
    2. "Repeat last week" — creates plan then copies previous week's slots
    3. "Load a template" — shows template list from useTemplates(), user picks one, then creates plan and loads template

    Modal overlay with bg-surface card, centered. Three buttons/cards for options. If "Load template" selected, show a secondary list of templates. Style: bg-background/80 backdrop, card with rounded-card border-accent/30.

    After user makes a choice, the plan row is created, which prevents the prompt from showing again this week.

    **TemplateManager.tsx:** Inline section on PlanPage (below week nav, above grid). Shows:
    - "Save as template" button — prompts for name (simple text input modal), calls useSaveAsTemplate
    - "Load template" button — shows template list, confirmation modal ("This will replace your current plan for all 7 days. Continue?" per pitfall 5), then calls useLoadTemplate
    - Template list shows name + created date + delete button
