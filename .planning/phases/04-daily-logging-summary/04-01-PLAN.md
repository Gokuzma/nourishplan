1. Create `supabase/migrations/009_food_logs.sql`:
   - `food_logs` table with columns: id (uuid PK), household_id (FK households), logged_by (FK auth.users), member_user_id (FK auth.users, nullable), member_profile_id (FK member_profiles, nullable), log_date (date NOT NULL), slot_name (text, nullable), meal_id (FK meals, nullable, ON DELETE SET NULL), item_type (text, nullable — 'food'|'recipe'|'meal'), item_id (text, nullable), item_name (text NOT NULL), servings_logged (numeric NOT NULL CHECK > 0), calories_per_serving (numeric NOT NULL), protein_per_serving (numeric NOT NULL), fat_per_serving (numeric NOT NULL), carbs_per_serving (numeric NOT NULL), micronutrients (jsonb DEFAULT '{}'), is_private (boolean DEFAULT false), created_at (timestamptz DEFAULT now()), updated_at (timestamptz DEFAULT now())
   - Check constraint: exactly one of member_user_id or member_profile_id must be non-null
   - Indexes on (household_id, log_date), (member_user_id, log_date), (member_profile_id, log_date)
   - set_updated_at trigger (reuse existing function)
   - Enable RLS with 4 policies using get_user_household_id() and get_user_household_role():
     - SELECT: household member, non-private OR own log OR admin
     - INSERT: household member, logged_by = auth.uid()
     - UPDATE: logged_by = auth.uid() OR admin
     - DELETE: logged_by = auth.uid() OR admin
   - See RESEARCH.md Pattern 1 and Pattern 2 for exact schema and RLS SQL
