<feature>
  <name>Portion Suggestion Algorithm</name>
  <files>src/utils/portionSuggestions.ts, tests/portionSuggestions.test.ts</files>
  <behavior>
    --- calcRemainingCalories ---
    - Given target of 2000 cal and logs totaling 800 cal -> returns 1200
    - Given target of 2000 cal and logs totaling 2500 cal -> returns 0 (clamped, never negative)
    - Given null target -> returns 0
    - Given empty logs array -> returns full target value

    --- calcPortionSuggestions ---
    - Given 2 members with remaining 1200 and 800 cal, dish of 1 serving:
      Member A gets 60% (0.6 svg), Member B gets 40% (0.4 svg)
    - Given 3 members where one has no target:
      Members with targets get proportional split; no-target member gets 1.0 serving with percentage null
    - Given all members at zero remaining cal:
      All members get 1.0 serving default (division by zero guard)
    - Leftover percentage: 100 - sum(member percentages) reported in result
    - Members sorted: current user first per locked decision

    --- hasMacroWarning ---
    - Given member with 100g protein target, 70g logged, suggested portion adds 45g protein:
      70 + 45 = 115g, target 100g, over by 15% -> no warning (under 20%)
    - Given member with 100g protein target, 80g logged, suggested portion adds 45g protein:
      80 + 45 = 125g, target 100g, over by 25% -> warning (over 20%)
    - Given member with 200g carbs target, 50g logged, suggested portion adds 20g carbs:
      50 + 20 = 70g, target 200g, under by 65% -> warning (under by >20% means remaining is >20% of target)
      Actually: warning triggers if the suggested portion would put them >20% OVER or UNDER for the DAY.
      Under-warning: if after eating this portion, remaining daily target for that macro is still >20% of target AND this is not the first meal, flag it. Simpler: check if total_logged + portion_macro > target * 1.2 (over) or total_logged + portion_macro < target * 0.8 AND there are no more meal slots remaining (under only matters at last meal).
      Per user decision: "if the suggested portion would put a member >20% over or under any macro target for the day" — so check: (logged + portion) > target * 1.2 OR (logged + portion) < target * 0.8. The "under" check applies to the cumulative day total after this portion.
    - No target for a macro -> no warning for that macro
    - Warning is true if ANY of protein/carbs/fat triggers

    --- Multi-dish slot distribution ---
    - Given a slot with 2 dishes (400 cal and 600 cal), member remaining 500 cal:
      Member's share of 400-cal dish = 500 * (400/1000) = 200 cal worth
      Member's share of 600-cal dish = 500 * (600/1000) = 300 cal worth
      Servings = proportional to dish total calories
  </behavior>
