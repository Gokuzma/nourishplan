<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Set up Vercel project, DNS, and Supabase auth</name>
  <action>
These steps require human interaction with external dashboards. Complete them in order:

**Step 1 — Vercel project:**
1. Go to vercel.com -> "Add New Project"
2. Import the GitHub repo created in Task 1
3. Framework Preset: Vite (should auto-detect)
4. Build Command: `tsc -b && vite build`
5. Output Directory: `dist`
6. Add environment variables:
   - `VITE_SUPABASE_URL` = your Supabase project URL
   - `VITE_SUPABASE_ANON_KEY` = your Supabase anon key
7. Click Deploy

**Step 2 — Custom domain:**
1. In Vercel project -> Settings -> Domains
2. Add `nourishplan.gregok.ca`
3. Vercel will show a CNAME target (likely `cname.vercel-dns.com`)

**Step 3 — DNS record:**
1. Go to your DNS registrar for gregok.ca
2. Add CNAME record: Name=`nourishplan`, Value=`cname.vercel-dns.com`, TTL=300
3. Wait a few minutes for propagation

**Step 4 — Supabase auth configuration:**
1. Supabase Dashboard -> Authentication -> URL Configuration
2. Set **Site URL** to `https://nourishplan.gregok.ca`
3. Add `https://nourishplan.gregok.ca/**` to **Redirect URLs**
4. Keep `http://localhost:5173/**` in Redirect URLs (for local dev)
5. Go to Authentication -> Auth Settings
6. Disable "Allow new users to sign up" (invite-only)
  </action>
