Create a single migration file with these changes:

1. **custom_foods.portions** — Add jsonb column for user-defined measurement units:
```sql
ALTER TABLE public.custom_foods
ADD COLUMN IF NOT EXISTS portions jsonb NOT NULL DEFAULT '[]'::jsonb;
```
Each entry: `{description: string, grams: number}` (e.g., `[{"description": "1 cup", "grams": 240}]`).

2. **nutrition_targets.macro_mode** — Add column tracking grams vs percent display preference:
```sql
ALTER TABLE public.nutrition_targets
ADD COLUMN IF NOT EXISTS macro_mode text NOT NULL DEFAULT 'grams'
  CHECK (macro_mode IN ('grams', 'percent'));
```

3. **Profiles RLS — household member visibility** — Current profiles SELECT policy only allows users to see their own row. Household members need to see each other's display_name and avatar_url. Add a new SELECT policy:
```sql
CREATE POLICY "household members can view profiles"
  ON public.profiles FOR SELECT
  TO authenticated
  USING (
    EXISTS (
      SELECT 1 FROM public.household_members hm1
      JOIN public.household_members hm2 ON hm1.household_id = hm2.household_id
      WHERE hm1.user_id = (SELECT auth.uid())
        AND hm2.user_id = profiles.id
    )
  );
```

4. **Households admin UPDATE policy** — Allow admins to update household name:
```sql
CREATE POLICY "admins update household"
  ON public.households FOR UPDATE
  TO authenticated
  USING (
    EXISTS (
      SELECT 1 FROM public.household_members
      WHERE household_id = households.id
        AND user_id = (SELECT auth.uid())
        AND role = 'admin'
    )
  )
  WITH CHECK (
    EXISTS (
      SELECT 1 FROM public.household_members
      WHERE household_id = households.id
        AND user_id = (SELECT auth.uid())
        AND role = 'admin'
    )
  );
```

5. **Profiles UPDATE policy** — Allow users to update their own profile (display_name, avatar_url):
```sql
CREATE POLICY "users update own profile"
  ON public.profiles FOR UPDATE
  TO authenticated
  USING ((SELECT auth.uid()) = id)
  WITH CHECK ((SELECT auth.uid()) = id);
```
