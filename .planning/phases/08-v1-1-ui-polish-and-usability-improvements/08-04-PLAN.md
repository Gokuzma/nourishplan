**PortionStepper enhancement:**

1. Export a `PortionUnit` type:
```ts
export interface PortionUnit {
  description: string  // "1 cup", "1 medium", "100g"
  grams: number        // gram equivalent
}
```

2. Add optional props to PortionStepperProps:
```ts
units?: PortionUnit[]           // available units from food data
selectedUnit?: PortionUnit      // currently selected unit
onUnitChange?: (unit: PortionUnit) => void
```

3. When `units` is provided and has items, render a `<select>` dropdown above the stepper row showing available units. Each option shows the unit description. Include a "grams" option as fallback with `{ description: "grams", grams: 1 }`.

4. The `value` prop now represents quantity in the selected unit (e.g., "2" cups, not 200 grams). The parent computes `totalGrams = value * selectedUnit.grams` for nutrition math.

5. When no units are provided (or empty array), render the stepper exactly as it does today — no dropdown, no behavior change. This ensures backward compatibility.

6. Adjust step size based on unit: for gram-based units use step=10, for household units (cups, pieces) use step=0.25 (current default).
