**Add a useEffect to RecipeBuilder that hydrates foodDataMap from persisted data when ingredients load.**

After the existing `const [foodDataMap, setFoodDataMap] = useState<Record<string, FoodDataEntry>>({})` line, add a `useEffect` that runs when `ingredients` changes:

```typescript
import { useState, useMemo, useEffect } from 'react'
// ... (useEffect already available if not imported)

// Inside RecipeBuilder component, after foodDataMap state declaration:
useEffect(() => {
  if (!ingredients || ingredients.length === 0) return

  // Find ingredient_ids that are NOT already in foodDataMap
  const missingIds = ingredients
    .filter(ing => ing.ingredient_type === 'food' && !foodDataMap[ing.ingredient_id])
    .map(ing => ing.ingredient_id)

  if (missingIds.length === 0) return

  // Batch-fetch custom food macros for missing ingredients
  async function hydrate() {
    const { data: customFoods } = await supabase
      .from('custom_foods')
      .select('id, name, calories_per_100g, protein_per_100g, fat_per_100g, carbs_per_100g, micronutrients')
      .in('id', missingIds)

    if (!customFoods || customFoods.length === 0) return

    const newEntries: Record<string, FoodDataEntry> = {}
    for (const f of customFoods) {
      newEntries[f.id] = {
        name: f.name,
        macros: {
          calories: f.calories_per_100g,
          protein: f.protein_per_100g,
          fat: f.fat_per_100g,
          carbs: f.carbs_per_100g,
        },
        micronutrients: f.micronutrients ?? null,
      }
    }

    setFoodDataMap(prev => ({ ...prev, ...newEntries }))
  }

  hydrate()
  // eslint-disable-next-line react-hooks/exhaustive-deps
}, [ingredients])
```

**Key details:**
- The dependency array includes only `ingredients` — NOT `foodDataMap` (would cause infinite loop). The `// eslint-disable-next-line` comment suppresses the exhaustive-deps warning for `foodDataMap` which is intentionally excluded.
- The `missingIds` filter checks `!foodDataMap[ing.ingredient_id]` using the current closure value. This means on first render (foodDataMap = {}), it fetches all custom food ingredients. After state updates, re-renders won't re-fetch because `ingredients` reference hasn't changed.
- For USDA/CNF ingredients (non-UUID IDs), they won't match any `custom_foods` rows, so those ingredients will still show 0 nutrition until re-searched. This is a known v1 limitation documented in the research.
- Recipe-type ingredients (`ingredient_type === 'recipe'`) are excluded from this hydration — they use `resolveRecipeNutrition` which already fetches from custom_foods internally.

**Also ensure `useEffect` is imported.** Check the import line at the top — currently `import { useState, useMemo } from 'react'`. Change to `import { useState, useMemo, useEffect } from 'react'`.
