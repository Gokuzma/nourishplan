## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (e.g. p-1, gap-1) |
| sm | 8px | Compact element spacing (e.g. p-2, gap-2) |
| md | 16px | Default element spacing (e.g. p-4, gap-4) |
| lg | 24px | Section padding (e.g. p-6, gap-6) |
| xl | 32px | Layout gaps (e.g. gap-8) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Inline delete confirmation text: renders in-place within existing list row padding (inherits row padding, no extra spacing token).
- Touch targets for delete/confirm buttons: minimum 44px height on mobile (p-2.5 at 16px base = 44px tap area).
- Danger Zone section: 32px top margin (`mt-8`) to visually separate from other settings content.

Source: existing component patterns in `LogEntryItem.tsx`, `MealCard.tsx`, `SettingsPage.tsx`.

## Typography

Two weights only: 400 (regular) and 600 (semibold). No 500/medium — replace any `font-medium` with `font-normal` (body/label) or `font-semibold` (heading/display).

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 400 (regular) | 1.4 |
| Heading | 16px | 600 (semibold) | 1.25 |
| Display | 20px | 600 (semibold) | 1.2 |

Notes:
- Body (14px/400) is the dominant type size — `text-sm font-normal` for item names, `text-xs text-text/50` for metadata. Source: `LogEntryItem.tsx:43–44`, `MealCard.tsx:28–29`.
- Recipe notes subtitle: 14px/400, `text-text/50` — renders below recipe name as a muted subtitle.
- Date created tag: 12px/400, `text-text/40` — inline with recipe servings info on list cards.
- "Created 3 days ago" text uses `text-xs text-text/40`; full date shown on hover via `title` attribute (no tooltip library).
- Danger Zone heading: 16px/600 `text-red-600` to signal destructive context.
- Print sizes (11px body, 13px heading) apply only in `@media print` browser context and are not part of the main type scale above. Print uses browser default system font; Nunito is not loaded in print.

## Interaction Contracts

### Inline Delete Confirmation

1. User taps delete icon (×) on a meal, recipe, or custom food row.
2. The × icon has `aria-label="Delete [item name]"` (e.g. `aria-label="Delete Chicken Stir Fry"`).
3. Delete icon is replaced in-place with: `Delete [item name]?  [Yes, delete]  [Keep it]`
4. Tapping "Yes, delete" triggers `useMutation` delete; tapping "Keep it" restores the delete icon.
5. No modal overlay. No separate confirmation page. Confirmation text is 12px, destructive red for "Yes, delete", primary muted for "Keep it".
6. State: `deleteConfirmId: string | null` in the parent list component — existing pattern in `FoodSearchOverlay.tsx`.

### Account Deletion — Admin with Other Members

1. Admin opens Danger Zone → taps "Delete my account".
2. Modal (full-screen on mobile, centered dialog on desktop) shows member list: "Before deleting, choose who becomes the new household admin."
3. Admin selects one member → taps "Transfer admin & continue".
4. Second step shows: input field with placeholder "Type DELETE to confirm", below: "This will permanently delete your account. The household and all data will remain."
5. Input must match "DELETE" exactly (case-sensitive) to enable the red "Delete my account" button.
6. On confirm: execute admin transfer then account deletion via Supabase RPC.

### Account Deletion — Last Member in Household

1. User opens Danger Zone → taps "Delete my account".
2. Single-step modal: "You are the only member. Deleting your account will permanently delete the [Household Name] household and all its data."
3. Input field: type "DELETE" to confirm. Red "Delete household and account" button enables on match.

### Account Deletion — Non-Admin Member

1. Member opens Danger Zone → taps "Delete my account".
2. Single-step modal: "This will permanently delete your account. The household and all its data will remain."
3. Input field: type "DELETE" to confirm. Red "Delete my account" button enables on match.

### Recipe Notes Field

1. `<textarea>` appears directly below recipe name input in RecipeBuilder.
2. Initial height: 1 line (approximately 40px). Grows on content via `rows` auto-adjust or `resize: none` + `overflow: hidden` + JS height update on input.
3. Placeholder: "Add notes or variations..."
4. Saves on blur via `useUpdateRecipe` mutation (not a separate save button).
5. Empty notes field: not shown on read-only recipe list cards.

### Date Created Display

1. On RecipesPage list cards: "Created 3 days ago" — `text-xs text-text/40` — appended after servings count with ` · ` separator.
2. Full date (e.g. "March 13, 2026") shown via `title` HTML attribute on the relative time text — no tooltip component needed.

### Meal Plan Start Date

1. Date input (`<input type="date">`) appears at the top of the new-week creation flow (inside `NewWeekPrompt` or equivalent).
2. Defaults to the Monday of the current week (existing `getWeekStart` UTC logic).
3. Follows the same input styling as `NutritionTargetsForm` — `border border-secondary rounded-[--radius-btn] px-3 py-2 text-sm`.

### Print Meal Plan

1. Print button placed in overflow/⋮ menu on PlanPage. Label: "Print meal plan".
2. On click: calls `window.print()`.
3. Print CSS (`@media print`): hides TabBar, Sidebar, OfflineBanner, all buttons, app chrome. Shows only: 7-column meal grid (days × meal slots) + a daily totals row (cal / P / C / F).
4. Colors: all background colors forced to white; all text forced to black. Borders: `1px solid #000` on grid cells. No color accents.
5. Font in print: browser default system font, 11px body, 13px headings. These sizes apply only within `@media print` and are separate from the main type scale.
