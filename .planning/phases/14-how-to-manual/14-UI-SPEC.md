## Interaction Contract

### Accordion Behavior

- **Single-open:** Only one section is open at a time. Opening a second section closes the first.
- **Toggle:** Tapping an open section header collapses it.
- **Visual open state:** Open header shows a chevron pointing up (▲ or rotate-180 via Tailwind); closed header shows chevron pointing down.
- **Section body visibility:** Use `hidden` / `block` class toggle (no animation). CSS `max-height` animation is explicitly avoided per RESEARCH.md Pitfall 2.
- **Section header tap target:** Full-width button spanning 100% of the container, minimum height 44px for touch accessibility.

### Deep-link Behavior

- On mount, read `window.location.hash` (e.g. `#recipes`).
- If hash matches a section `id`, set that section open and call `element.scrollIntoView({ behavior: 'smooth' })`.
- Hash must be set **before** scroll is called (state update first, then effect).
- Section IDs: `getting-started`, `adding-foods`, `recipes`, `meal-plan`, `tracking`, `household-admin`.

**Source:** RESEARCH.md Pattern 2 and Pitfall 3.

### Navigation Item Placement

- In both MobileDrawer and Sidebar: "User Guide" appears **after "Settings"** — last item in the nav list.
- Uses emoji 📘 as icon, label "User Guide", route `/guide`.
- Active state follows existing `NavLink` pattern (same `className` function already used for all other nav items).

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Accordion headers are focusable | Use `<button>` element, not `<div>` |
| Accordion open/closed state | `aria-expanded="true/false"` on header button |
| Accordion body associated with header | `aria-controls="{section-id}-body"` on button; `id="{section-id}-body"` on body `<div>` |
| Deep-link target scrollable | `id` attribute on section wrapper `<div>`, not inside hidden body |
| Touch targets | Minimum 44px height on all interactive elements (accordion headers) |
| Color contrast | `text-text` on `bg-background` meets WCAG AA at both light (#3D3D3D on #FAFAF7) and dark (#E8E5E0 on #1A1D1A) |
