## Color

All tokens are from `src/styles/global.css` @theme block with dark mode overrides in `.dark {}` layer.

| Role | Value (light) | Value (dark) | Usage |
|------|--------------|--------------|-------|
| Dominant (60%) | `--color-background: #FAFAF7` | `#1A1D1A` | Page background |
| Secondary (30%) | `--color-surface: #FFFFFF` | `#252825` | Cards, modals, collapsible sections |
| Primary | `--color-primary: #A8C5A0` | `#B8D4B0` | Progress rings, positive indicators |
| Accent (10%) | `--color-accent: #E8B4A2` | `#F0C4B2` | Reserved list below |
| Destructive | `red-500` / `red-50` bg | same | Destructive actions (remove buttons) only |
| Text | `--color-text: #3D3D3D` | `#E8E5E0` | All body text |

Accent (`--color-accent`) reserved for:
- Budget section border and divider lines (`border-accent/20`, `border-accent/30`)
- Modal close button backgrounds
- Slot card borders on filled slots
- Price input focus ring

Cost/budget semantic colours (new in this phase):
- **Budget healthy** (spend < 80% of budget): `--color-primary` (sage green) — matches nutrition "on track" visual language
- **Budget warning** (spend 80–100% of budget): `amber-400` — matches existing macro warning pill colour in SlotCard
- **Budget over** (spend > 100% of budget): `red-500` — matches destructive/remove button colour

*Source: `src/styles/global.css`, `src/components/plan/SlotCard.tsx`, CONTEXT.md D-01 through D-05.*

## Component Inventory

New components introduced in this phase and their visual contracts:

### BudgetSummarySection (Plan page, below plan grid)

- Container: `rounded-xl border border-accent/20 bg-surface p-4`
- Collapsible: chevron toggle button (14×14 SVG, inline pattern matching SlotCard expand button)
- Default state: expanded
- Layout: two rows
  - Row 1: "Weekly Budget" label (text-sm text-text/60) + inline editable amount + "Edit" link
  - Row 2: spend bar + "spent $X.XX of $Y.YY · $Z.ZZ remaining" (text-xs text-text/50)
- Spend bar: full-width `h-2 rounded-full bg-secondary` track, `bg-primary` fill (healthy), `bg-amber-400` fill (warning), `bg-red-500` fill (over)
- Per-day cost chips: `flex gap-2 flex-wrap mt-2`, each chip `text-xs px-2 py-0.5 rounded-full bg-background border border-accent/20`

### CostBadge (recipe cards and recipe detail subtitle line)

- Placement: subtitle line below recipe name, before serving count
- Format: `$2.45/serving · 4 servings` (text-xs text-text/50)
- Partial: `$X.XX+/serving · (3 of 5 priced)` — same style, `+` suffix signals incomplete
- No separate component required — inline `<p className="text-xs text-text/50 font-sans">` matching existing SlotCard subtitle pattern

### InlinePriceEntry (IngredientRow extension)

- Trigger: "Set price" link shown when ingredient has no price — `text-xs text-primary underline`
- Input row: appears inline below ingredient row, `py-2 px-3 border-t border-accent/10`
- Input fields: two adjacent inputs — "Amount ($)" + "for (qty + unit)" — `text-sm border border-accent/30 rounded-lg px-2 py-1 w-24`
- Save button: `text-xs px-3 py-1 rounded-lg bg-primary/10 text-primary hover:bg-primary/20`
- Cancel: `text-xs text-text/40 hover:text-text ml-2`

### FoodPricesSection (Settings page, new section after Household)

- Section header: matches existing Settings section pattern — `text-base font-semibold text-text mb-3`
- List: rows with food name, price, store, edit/delete inline
- Row: `flex items-center justify-between py-2.5 border-b border-accent/10`
- Missing price indicator: amber dot `w-2 h-2 rounded-full bg-amber-400` before food name
- "Add price" button: `text-sm px-3 py-1.5 rounded-lg bg-primary/10 text-primary`

### CookButton (recipe detail view)

- Placement: primary action area of recipe detail, alongside existing save/edit controls
- Label: "Mark as Cooked"
- Style: `px-4 py-2 rounded-lg bg-primary text-white text-sm font-medium hover:bg-primary/90`
- Post-tap state: brief "Cooked — spend recorded" toast-style inline confirmation (text-xs text-primary, fades after 2s) — no modal required

## Interaction Contracts

### Budget Inline Edit (Plan page)

- "Edit" link on the budget section triggers an inline amount input replacing the displayed value
- Input: `type="number" min="0" step="0.01"` — CAD $ prefix label beside it
- Save on blur or Enter key; cancel on Escape
- No modal — matches inline editing pattern from household name in SettingsPage

### Price Entry Format (InlinePriceEntry)

- Two input pattern: amount field + "for" quantity+unit field
- Examples: "$4.50" + "per kg" OR "$7.99" + "for 900g"
- System normalises to `cost_per_100g` on save — user never sees the normalised value directly
- Store selector: text input with `list` attribute pointing to a `<datalist>` of existing stores for the household (no forced selection — freeform supported)

### Cook Button

- Single tap records spend; no confirmation required (non-destructive)
- If recipe has partial pricing, still records; confirmation copy adjusts: "Cooked — partial spend recorded ($X.XX of estimated total)"
- Button disabled while mutation is in flight; shows loading spinner (14px ring, matching existing async patterns)

### Budget Progress Bar

- Animates fill on mount (CSS transition, 300ms ease-out)
- Color transition: green → amber at 80%, amber → red at 100%
- ARIA: `role="progressbar" aria-valuenow="{spend}" aria-valuemax="{budget}" aria-label="Weekly spend"`
