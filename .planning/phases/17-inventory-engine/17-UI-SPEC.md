## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline chip gaps |
| sm | 8px | Compact row padding, item meta spacing |
| md | 16px | Default card padding, section padding |
| lg | 24px | Page header padding, section vertical gaps |
| xl | 32px | Major section breaks |
| 2xl | 48px | Empty state vertical centering |
| 3xl | 64px | Bottom safe area for fixed tab bar |

Exceptions:
- Touch targets for row action buttons (Edit, Delete, Mark used): minimum 44px height
- Barcode scanner viewfinder: full-width, no horizontal padding (extends to screen edges)

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FAFAF7` light / `#1A1D1A` dark (`--color-background`) | Page background, inventory list background |
| Secondary (30%) | `#FFFFFF` light / `#252825` dark (`--color-surface`) | Item row cards, tab panels, modal surfaces |
| Accent (10%) | `#E8B4A2` light / `#F0C4B2` dark (`--color-accent`) | See reserved list below |
| Destructive | `#DC2626` (Tailwind red-600) | "Remove item" destructive action only |

**Accent reserved for:**
- Expiry urgency badge borders and text (items expiring within 3 days)
- Active tab indicator underline on Pantry / Fridge / Freezer tab strip
- "Add item" primary CTA button background (consistent with existing primary button pattern using `--color-primary` sage green `#A8C5A0`)

**Note on primary CTA:** The "Add item" button uses `--color-primary` (#A8C5A0 sage green), consistent with every other primary action button in the app (RecipesPage, HomePage). Accent (#E8B4A2 peach) is reserved for urgency and active state signals only.

**Expiry color semantics (D-19 — Claude's discretion):**
- Expired: `bg-red-100 text-red-700` (light) / `bg-red-900/30 text-red-400` (dark) — solid badge
- Expiring within 3 days: `bg-accent/20 text-accent` pill badge — uses theme accent token
- Expiring within 7 days: `bg-primary/20 text-primary` pill badge — soft warning
- No expiry date: no badge shown

## Layout Patterns

### Inventory Page

```
┌─────────────────────────────────────┐
│  Inventory          [Scan] [+ Add]  │  ← page header, px-4 py-6
│  Manage your pantry, fridge, and    │
│  freezer stock.                     │
├─────────────────────────────────────┤
│  [Pantry]  [Fridge]  [Freezer]      │  ← location tab strip, border-b
│  ─────                              │    active tab: border-b-2 border-accent
├─────────────────────────────────────┤
│  🧅 Onions          200g  📦        │  ← InventoryItemRow
│     Expires in 5 days               │    bg-surface, rounded-[--radius-card]
│─────────────────────────────────────│    border border-secondary
│  🥚 Eggs            6 units  📦     │
│     [Expires Mar 28]  ← accent badge│
│─────────────────────────────────────│
│  🧈 Butter          250g  📦        │
│     Expired Mar 20  ← red badge     │
└─────────────────────────────────────┘
       [Home]  [Recipes]  [Plan]  [☰]   ← existing TabBar
```

**Tab strip:** Three tabs — Pantry, Fridge, Freezer. Implemented as `flex border-b border-secondary` with each tab button using `pb-2 border-b-2`. Active tab: `border-accent text-text font-medium`. Inactive: `border-transparent text-text/50`.

**Item sort order within each tab:** Expiring soonest first (INVT-03). Past-expiry items at top with red badge.

### InventoryItemRow (D-20 — inline edit via tap-to-expand)

Matches `LogEntryItem` expand pattern exactly:

```
[chevron]  [name]  [qty + unit]       [expiry badge]
           [brand? · location]        [actions on expand]
```

- Tap row: expands to show Edit button + Remove button (same stopPropagation pattern as LogEntryItem)
- Edit: opens `AddInventoryItemModal` pre-filled
- Remove: opens inline confirmation row below (matches Phase 13 delete pattern: `-mt` offset, `border-t-0`)
- Staple items: small "Staple" label badge in `bg-primary/10 text-primary` at end of name line

**Decision basis:** D-20 — inline editing via tap-to-expand matches the existing LogEntryItem and RecipesPage inline-delete patterns. No new interaction paradigm needed.

### AddInventoryItemModal

Bottom sheet (mobile) / centered modal (sm+). Matches `EditLogModal` pattern:

```css
fixed inset-0 z-50 flex items-end sm:items-center justify-center
```

Fields in order:
1. Item name (text input — pre-filled from barcode/food-search if available)
2. Brand (optional text input)
3. Quantity + unit selector (number input + dropdown: g / kg / ml / L / units)
4. Storage location (3-button toggle: Pantry | Fridge | Freezer)
5. Expiry date (date input, optional)
6. Purchase price (number input, optional — with nudge label "Add price for cost tracking")
7. Staple toggle (checkbox: "Mark as staple for restock alerts")

Submit: "Add to Inventory" primary button. Cancel: "Dismiss" ghost text link (matches existing modal pattern).

### BarcodeScanner Overlay

Full-screen overlay (`fixed inset-0 z-50 bg-black`):
- Camera viewfinder: centered square cutout with corner guides in `--color-primary` sage green
- Instructions text: `text-white/80 text-sm text-center` below viewfinder
- "Enter barcode manually" fallback: text input + "Look up" button in `bg-surface` panel at bottom
- Close: "✕" button top-right, `text-white text-xl`

### Home Page Widget (InventorySummaryWidget)

Inserted into `HomePage.tsx` as a card section, consistent with the existing daily-progress card pattern:

```
┌─────────────────────────────────────┐
│  Inventory                          │  ← section heading text-sm font-medium
│  Fridge: 12  ·  Freezer: 8  ·      │  ← location count row, text-xs text-text/60
│  Pantry: 15                         │
│─────────────────────────────────────│
│  Expiring soon:                     │  ← text-xs text-text/50, only shown if items
│  • Chicken breast  (Mar 28)  [→]    │
│  • Eggs            (Mar 29)  [→]    │
│  • Butter          (Mar 30)  [→]    │
└─────────────────────────────────────┘
```

- Card: `bg-surface rounded-[--radius-card] border border-secondary p-4`
- Max 5 items in expiring-soon list
- "[→]" link taps to Inventory page (no per-item navigation at widget level)
- If no expiring items within 7 days: omit the expiring-soon sub-section entirely (no empty state in widget)

### CookDeductionReceipt

Shown as a toast notification pattern after Cook is pressed. Because the receipt may list multiple items, use a temporary bottom-anchored panel (not a standard toast):

```
┌─────────────────────────────────────┐
│  Cooked: Chicken Stir Fry           │  ← meal name
│  Deducted from inventory:           │
│  ✓ Chicken breast 200g (Mar 20 pkg) │
│  ✓ Broccoli 150g (Mar 22 pkg)       │
│  ⚠ Salt — not in inventory          │  ← missing items in text/50
│  ⚠ Olive oil — not in inventory     │
│                            [Done]   │
└─────────────────────────────────────┘
```

- Panel: `fixed bottom-16 md:bottom-4 inset-x-0 mx-4 bg-surface border border-secondary rounded-[--radius-card] p-4 shadow-xl z-50`
- Auto-dismiss after 8 seconds or on "Done" tap
- Missing items shown with `text-text/50` and `⚠` prefix (D-28)
- FIFO source shown in parentheses: "(Mar 20 pkg)" references `purchased_at` date (D-26)

## Interaction States

### Inventory Item Row States

| State | Visual |
|-------|--------|
| Default | `bg-surface border-secondary` |
| Hover (desktop) | `border-accent/40` (matches RecipesPage hover) |
| Expanded | chevron rotates 90deg, actions row below slides in |

### Tab Strip States

| State | Visual |
|-------|--------|
| Active | `border-b-2 border-accent text-text font-medium` |
| Inactive | `border-b-2 border-transparent text-text/50` |
| Hover | `text-text` |

### Barcode Scan States

| State | Visual |
|-------|--------|
| Scanning | Animated corner guides pulse in `--color-primary` |
| Item found | Guides flash green, `text-white` success caption below |
| Not found | Guides flash red, error copy shown, manual entry auto-focused |

## Accessibility

- All item rows: `role="button"` `aria-expanded={expanded}` `aria-label="{item name} inventory entry"` — mirrors LogEntryItem pattern
- Barcode scanner: `aria-live="polite"` region for scan status announcements
- Location tab strip: `role="tablist"` with each tab `role="tab"` `aria-selected={active}`
- Tab panel: `role="tabpanel"` with `aria-labelledby` pointing to active tab
- Remove confirm: focus management — focus moves to "Confirm" button when confirm row appears
- Touch targets: all interactive elements minimum 44px height (exception noted in spacing scale)
