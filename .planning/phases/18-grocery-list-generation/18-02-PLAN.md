**src/hooks/useGroceryItems.ts:**

```typescript
export function useGroceryItems(listId: string | undefined)
```
- Query: `supabase.from('grocery_items').select('*').eq('list_id', listId).order('category').order('food_name')`
- Key: `queryKeys.grocery.items(listId)`
- Enabled: `!!listId`
- **Supabase realtime subscription** (FIRST USE IN PROJECT): Set up a `useEffect` that creates a Supabase channel subscribed to `postgres_changes` on `grocery_items` table filtered by `list_id=eq.${listId}`. On any UPDATE/INSERT/DELETE event, invalidate `queryKeys.grocery.items(listId)`. Clean up with `supabase.removeChannel(channel)` on unmount. Channel name: `grocery-items-${listId}`.

```typescript
useEffect(() => {
  if (!listId) return
  const channel = supabase
    .channel(`grocery-items-${listId}`)
    .on('postgres_changes', {
      event: '*',
      schema: 'public',
      table: 'grocery_items',
      filter: `list_id=eq.${listId}`,
    }, () => {
      queryClient.invalidateQueries({ queryKey: queryKeys.grocery.items(listId) })
    })
    .subscribe()
  return () => { supabase.removeChannel(channel) }
}, [listId, queryClient])
```

**src/hooks/useGenerateGroceryList.ts:**

```typescript
export function useGenerateGroceryList()
```
- This is the orchestration mutation. It:
  1. Fetches current meal plan via `supabase.from('meal_plans')...` for the current week
  2. Fetches meal_plan_slots with nested `meals(*, meal_items(*))` for that plan
  3. For each recipe-type meal_item, fetches `recipe_ingredients` (and recursively for nested recipes)
  4. Fetches spend_logs with `source='cook'` for the current week to build `cookedRecipeIds` set (D-03)
  5. Fetches inventory items via `supabase.from('inventory_items')...`
  6. Fetches food prices via `supabase.from('food_prices')...`
  7. Fetches any previous grocery_items for this household to get user category overrides
  8. Calls `aggregateIngredients(slots, cookedRecipeIds)`
  9. Calls `subtractInventory(aggregated, inventoryItems)`
  10. Calls `assignCategories(needToBuy, previousItems)`
  11. Calls `addRestockStaples(existingFoodIds, inventoryItems)` and merges into needToBuy
  12. Computes `estimated_cost` for each item using `computeItemCost`
  13. Calls `formatDisplayQuantity` for each item to get display unit/quantity
  14. Upserts `grocery_lists` row: `supabase.from('grocery_lists').upsert({ household_id, week_start, generated_by: userId, generated_at: new Date().toISOString() }, { onConflict: 'household_id,week_start' }).select().single()`
  15. Deletes existing non-manual items: `supabase.from('grocery_items').delete().eq('list_id', listId).eq('is_manual', false)`
  16. Inserts all generated items (needToBuy + alreadyHave marked differently) as grocery_items rows
  17. Also inserts "Already Have" items with a flag (they have `is_checked = false`, quantity shows the covered amount, can use `notes` field like "In inventory")
- On success: invalidate `['grocery', householdId]` prefix to refresh all grocery queries

Implementation note: The "Already Have" items should be stored in the DB with a distinguishing field. Use `notes = 'inventory'` to mark them as inventory-covered items. The UI will filter by this to show in the "Already Have" section. Alternatively, create a separate boolean `is_inventory_covered` — but since the migration doesn't have this column, use the `notes` field as a marker: items with `notes = 'inventory'` go in Already Have section.

Actually, simpler approach: Do NOT store "Already Have" items in the DB at all. Compute them client-side by comparing the generated list against current inventory on each render. This avoids stale inventory snapshots and keeps the grocery_items table clean. The hooks should:
- `useGroceryItems` returns DB items (all "Need to Buy" + manual items)
- The GroceryPage computes "Already Have" items by running subtractInventory on the original plan data against current inventory — or even simpler, store the "Already Have" items in the DB too for snapshot consistency per D-01.

Final decision: Store both in DB for snapshot consistency (D-01 says "static until regenerated"). Use `notes = 'inventory-covered'` on Already Have items. The UI filters: `items.filter(i => i.notes !== 'inventory-covered')` for Need to Buy, `items.filter(i => i.notes === 'inventory-covered')` for Already Have.
