## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, ring gaps (gap-1, gap-0.5) |
| sm | 8px | Compact element spacing, button padding (p-2, gap-2) |
| md | 16px | Default element spacing, card padding (p-4) |
| lg | 24px | Section padding (py-6) |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level bottom padding (pb-[64px]) |

Exceptions:
- Drag handle touch target: 44px minimum height (WCAG 2.5.5 touch target) — handle zone is `min-h-[44px]` on mobile even if visual element is smaller
- Drop action menu items: 44px minimum height on mobile touch targets
- Carousel day card peek: 16px visible edge on adjacent days (peek amount)

Source: existing codebase uses `p-4`, `gap-2`, `gap-4`, `py-2 px-3` on slot cards, `pb-[64px]` on pages.

## Interaction Contract

### Drag Initiation (CONTEXT.md D-01, D-02)

- Drag ONLY initiates from the DragHandle (≡ icon). Touching the card body does not start a drag.
- On drag start: source SlotCard scales down to 0.95 and fades to opacity-50, showing a dashed placeholder outline in its place.
- The lifted card renders at 100% scale with `shadow-xl` and a sage green (`primary/20`) box shadow halo.
- Lift animation: `transition-transform duration-150 ease-out` scale from 1.0 to 1.05 on pickup.
- Drop animation: `transition-transform duration-200 ease-in` scale from 1.05 to 1.0 on release.

### Drop onto Empty Slot (CONTEXT.md D-11)

- The empty slot highlights with `border-primary/60 bg-primary/5` while the dragged card is over it.
- On drop: meal moves to target slot; source slot clears. No confirmation needed.
- Visual feedback during hover: border transitions from `border-dashed border-accent/30` to `border-solid border-primary/60`.

### Drop onto Occupied Slot (CONTEXT.md D-10)

- The DropActionMenu appears anchored below the target SlotCard.
- Two buttons: "Swap" (exchanges both meals) and "Replace" (moves dragged meal in, clears source, discards target meal).
- Menu auto-dismisses if user taps anywhere outside it or presses Escape.
- "Replace" is the destructive option — button text uses `text-red-500`, no separate confirmation dialog.

### Locked Slot Visual Treatment (CONTEXT.md D-04, D-05, D-06)

- Locked slot: 3px solid left border in `primary` color (#A8C5A0 / #B8D4B0 dark). Applied as `border-l-[3px] border-l-primary`.
- Lock badge: 16x16px rounded square in top-right corner of the SlotCard. Lock SVG icon (closed padlock). Filled `bg-primary text-white` when locked; `bg-surface border border-accent/30 text-text/40` (outline) when unlocked.
- Locked badge is always visible on occupied slots (toggle affordance). Not shown on empty slots.
- Dragging a locked slot: allowed — drag handle is still active. The lock only gates auto-generation (Phase 22).

### Mobile Carousel (CONTEXT.md D-08, D-09)

- Layout switches at the `md` breakpoint (768px), identical to existing `md:hidden` / `hidden md:flex` split.
- Carousel uses `overflow-x-auto scroll-snap-type-x mandatory` with each DayCard as a `scroll-snap-align-center` item.
- Card width: `calc(100vw - 48px)` with 16px left/right margins, leaving adjacent days 16px peeking from each edge.
- Days do NOT automatically scroll during cross-day drag — user drags onto a visible adjacent peek area to trigger a drop on that day.
- Current day indicator dots remain at top (existing pattern, 7 dots). Active dot: `bg-primary`. Inactive: `bg-accent/40`.

### Auto-scroll (Claude's Discretion)

- Desktop drag: auto-scroll activates when the dragged card is within 80px of the viewport top or bottom edge. Scroll speed: 6px per animation frame.
- Mobile carousel: no auto-scroll — user drags onto the visible peek of adjacent days.

### dnd-kit Configuration

- Library: `@dnd-kit/core` + `@dnd-kit/sortable` (confirmed in STATE.md v2.0 decisions).
- Sensors: `PointerSensor` with `activationConstraint: { distance: 8 }` for desktop. `TouchSensor` with `activationConstraint: { delay: 0, tolerance: 5 }` for mobile — zero delay because drag handle isolates the interaction from scroll.
- DragOverlay: renders a clone of SlotCard at pointer position during drag, styled with `shadow-xl scale-105 opacity-95`.

## Accessibility Notes

- Drag handle must have `role="button"` and `tabIndex={0}` — keyboard users activate with Enter/Space to enter drag mode, then use arrow keys to navigate slots.
- `aria-grabbed` attribute on the dragged SlotCard during active drag.
- `aria-dropeffect="move"` on valid drop targets while drag is active.
- Lock badge toggle: `role="button"`, `aria-pressed={isLocked}`.
- DropActionMenu: `role="dialog"`, `aria-modal="true"`, focus trapped inside while open.
- Color is never the sole differentiator — locked state uses both border AND icon, drag source uses both opacity AND dashed outline.
