Create `src/hooks/useRatings.ts` with two hooks:

**useUnratedCookedMeals(householdId, userId, today):**
- Uses `useQuery` with a compound queryFn:
  1. Query `spend_logs` WHERE `household_id = householdId` AND `logged_by = userId` AND `log_date = today` AND `source = 'cook'` AND `recipe_id IS NOT NULL`. Select `recipe_id`.
  2. If no results, return empty array.
  3. Extract unique recipeIds from spend_logs results.
  4. Query `recipe_ratings` WHERE `rated_by_user_id = userId` AND `rated_at = today` AND `recipe_id IN (recipeIds)`. Select `recipe_id`.
  5. Filter out already-rated recipe IDs.
  6. Query `recipes` WHERE `id IN (unratedIds)`. Select `id, name`.
  7. Return `{ recipeId: string, recipeName: string }[]`.
- Query key: `['unrated-cooked', householdId, userId, today]`
- `enabled: !!householdId && !!userId && !!today`

**useRateMeal():**
- Uses `useMutation` that accepts `{ householdId, recipeId, recipeName, rating, userId }`.
- Inserts into `recipe_ratings`: `{ household_id, recipe_id, recipe_name, rated_by_user_id: userId, rating, rated_at: today }` where today is computed as `new Date().toISOString().slice(0, 10)`.
- On success: invalidate `['unrated-cooked']` prefix and `['ratings', householdId]` prefix to refresh both the card and any ratings lists.
- Uses `useQueryClient()` from TanStack Query for invalidation.

Import `supabase` from `../lib/supabaseClient`, `useQuery`/`useMutation`/`useQueryClient` from `@tanstack/react-query`, `queryKeys` from `../lib/queryKeys`.

Export both hooks as named exports.
