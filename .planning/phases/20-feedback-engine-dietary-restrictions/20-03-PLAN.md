1. Create `src/hooks/useDietaryRestrictions.ts`:

**useRestrictions(householdId, memberId, memberType: 'user' | 'profile'):**
- `useQuery` with `queryKey: queryKeys.restrictions.forMember(householdId, memberId)`
- queryFn: query `dietary_restrictions` table filtering by `member_user_id = memberId` when `memberType === 'user'`, or `member_profile_id = memberId` when `memberType === 'profile'`
- Returns single row (one restriction record per member) via `.maybeSingle()`
- `enabled: !!householdId && !!memberId`

**useSaveRestrictions():**
- `useMutation` accepting `{ householdId, memberId, memberType: 'user' | 'profile', predefined: string[], customEntries: string[] }`
- Upserts to `dietary_restrictions`: build the row with `household_id`, `member_user_id` or `member_profile_id` (based on memberType), `predefined`, `custom_entries`, `updated_at: new Date().toISOString()`
- Use `.upsert()` with `onConflict` matching the unique index on member
- On success:
  1. Invalidate `['restrictions', householdId]` prefix
  2. Fire-and-forget: `supabase.functions.invoke('classify-restrictions', { body: { memberId, memberType, restrictions: predefined, customEntries, householdId } })` — do NOT await the result; catch errors silently
- Return mutation object

**useWontEat hooks — create `src/hooks/useWontEat.ts`:**

**useWontEatEntries(householdId, memberId, memberType: 'user' | 'profile'):**
- `useQuery` with `queryKey: queryKeys.wontEat.forMember(householdId, memberId)`
- queryFn: query `wont_eat_entries` filtering by member (same pattern as restrictions) ordered by `created_at desc`
- `enabled: !!householdId && !!memberId`

**useAddWontEat():**
- `useMutation` accepting `{ householdId, memberId, memberType, foodName: string }`
- Inserts into `wont_eat_entries`: `{ household_id, member_user_id or member_profile_id, food_name, strength: 'dislikes', source: 'manual' }`
- On success: invalidate `['wont-eat', householdId]` prefix

**useUpdateWontEatStrength():**
- `useMutation` accepting `{ id: string, strength: 'dislikes' | 'refuses' | 'allergy', householdId }`
- Updates `wont_eat_entries` SET `strength = strength` WHERE `id = id`
- On success: invalidate `['wont-eat', householdId]` prefix

**useRemoveWontEat():**
- `useMutation` accepting `{ id: string, householdId }`
- Deletes from `wont_eat_entries` WHERE `id = id`
- On success: invalidate `['wont-eat', householdId]` prefix
