1. Create `src/hooks/usePlanViolations.ts`:

```typescript
export interface SlotViolation {
  slotId: string
  dayIndex: number
  mealType: string
  recipeName: string
  memberName: string
  memberId: string
  foodName: string
  strength: 'dislikes' | 'refuses' | 'allergy'
}
```

**usePlanViolations(householdId, slots, members, memberProfiles):**
- Uses `useQuery` to fetch all won't-eat entries for the household: query `wont_eat_entries` WHERE `household_id = householdId`, select all
- `enabled: !!householdId`
- In a `useMemo`, compute violations by cross-referencing:
  - For each slot with a meal: get the meal's `meal_items` (which have ingredient names)
  - For each meal item: check if its `food_name` or ingredient name appears in any member's won't-eat entries (case-insensitive substring match using `.toLowerCase().includes()`)
  - If match: create a `SlotViolation` with the slot details, member name, food name, and strength
- Return `{ violations: SlotViolation[], hasAllergyViolation: boolean }` where `hasAllergyViolation = violations.some(v => v.strength === 'allergy')`
- Note: the matching is intentionally simple (substring) because AI resolves exact ingredient names into the won't-eat list — the planner does not need fuzzy matching here

2. Create `src/hooks/useMonotonyWarnings.ts`:

**useMonotonyWarnings(slots, currentWeekStart, priorWeekSlots):**
- Pure computation hook (no query — data comes from PlanPage level)
- Uses `useMemo` calling `detectMonotony()` from `src/utils/monotonyDetection.ts`
- Converts meal_plan_slots to `SlotEntry[]` format: extract `recipeId` from slot's meal_items (get the recipe references — each slot's meal can have meal items that reference recipes)
- Returns `MonotonyWarning[]`
- Note: the executor needs to inspect how slots currently carry recipe references. Meal plan slots have `meal_id` and the joined meal has `meal_items` which reference foods/recipes. For monotony, we care about `meal_id` uniqueness or the meal's underlying recipe_id if it's a single-recipe meal. Use the meal name or recipe references from meal_items to construct SlotEntry objects.
