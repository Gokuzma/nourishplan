## Design System

| Property | Value |
|----------|-------|

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, star rating gaps, badge inner padding |
| sm | 8px | Compact element spacing, between inline tags |
| md | 16px | Default card/panel padding, form row spacing |
| lg | 24px | Section padding (matches existing `px-4 py-6` pattern) |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level bottom spacing (matches `pb-[64px]` pattern) |

Exceptions:
- Touch targets for rating stars: minimum 44px tall hit area (WCAG 2.5.5). Render stars at 24px visual, pad tap zone to 44px.
- Restriction checkbox rows: 44px min height for reliable touch interaction.
- Issues panel collapsed state: 48px tap target for the toggle header.

## Color

Source: `src/styles/global.css` `@theme` tokens. Light and dark values both declared below.

| Role | Light Value | Dark Value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `#FAFAF7` (`--color-background`) | `#1A1D1A` | Page backgrounds, panel backgrounds |
| Secondary (30%) | `#F5EDE3` (`--color-secondary`) | `#2A2E2A` | Cards, the Issues panel container, dietary restriction section background, rating card background |
| Accent (10%) | `#E8B4A2` (`--color-accent`) | `#F0C4B2` | Reserved for specific elements listed below |
| Primary (interactive) | `#A8C5A0` (`--color-primary`) | `#B8D4B0` | All actionable / positive interactive elements |
| Destructive | `#EF4444` (red-500, Tailwind) | `#EF4444` | Hard-block allergy override confirmation only |

Accent (`--color-accent` / `#E8B4A2`) is reserved for:
- Border on the "Rate today's meals" card on HomePage
- AI insight tag pill borders and text on recipe cards (e.g., "crowd-pleaser", "divisive")
- Issues panel warning indicators (border-left stripe on conflict rows)

Primary (`--color-primary` / `#A8C5A0`) is used for:
- Active (filled) star rating indicators
- Satisfied button active state
- Restriction checkbox checked state
- Confirm action buttons ("Save restrictions", "Add to won't-eat list")
- Won't-eat entry strength badge for "allergy" (hard block) — dark red border overrides for allergen tier

Semantic strength colors for won't-eat preference tiers:
- "dislikes" strength: `text-text/50` + `border-accent/40` (muted, informational)
- "refuses" strength: `text-amber-600` + `border-amber-300` (soft block, warning amber — matches existing macro warning pattern)
- "allergy" strength: `text-red-600` + `border-red-300` (hard block)

## Interaction Contracts

### Rating Card Lifecycle

1. After any meal is cooked (via Cook button flow), the RateMealsCard appears on HomePage.
2. Card remains until the member has rated every cooked meal for the current day.
3. Tapping a star on a MealRatingRow immediately saves the rating (no extra confirm).
4. Row shows `text-xs text-primary` "Saved" inline after submit — no toast.
5. When all rows are rated, the entire RateMealsCard fades out (`transition-opacity duration-300`).
6. If the member skips rating (navigates away), card reappears on next HomePage load while unrated meals exist for that day.

### Restriction Save Flow

1. User taps checkboxes and/or adds custom entries.
2. Tapping "Save restrictions" saves to DB immediately.
3. AI mapping Edge Function fires asynchronously — UI shows "Mapping ingredients in the background…" text for up to 5s, then hides.
4. No loading spinner on save button — optimistic save assumed instant, AI is fire-and-forget from the user's perspective.

### Won't-Eat Add Flow

1. User types food name in text input, taps "Add".
2. Entry appears immediately in list with default strength "Dislikes".
3. User can change strength inline via segmented control — change saves on tap (no confirm).
4. Remove entry: tap `×` — no confirmation dialog (not destructive enough to warrant it).

### Allergy Hard-Block Override

When a user attempts to add a meal containing a hard-blocked (allergy) ingredient to a plan slot:
- Block prompt: `window.confirm` dialog — "This meal contains [ingredient], which is flagged as an allergy for [member]. Add anyway?"
- If confirmed: slot is assigned with a persistent red warning badge.
- If declined: meal picker remains open.

(Uses `window.confirm` matching the existing calorie recalculation precedent set in Phase 8.)

### Issues Panel

- Default state on Plan page: collapsed.
- Auto-expands to show issues if any allergy (hard block) violation exists in the current week.
- Soft conflicts (dislikes, refuses, monotony) do not auto-expand — badge count on collapsed header is sufficient.
- Swap suggestion tap: confirms swap via existing DropActionMenu pattern (no separate dialog).

## Accessibility

- Star rating: each star is a `<button>` with `aria-label="Rate [meal name] [N] out of 5 stars"`
- Strength segmented control: `role="group"` with `aria-label="Preference strength"`, each option is a `<button aria-pressed>`
- Issues panel toggle: `aria-expanded` on the header button
- IssueRow: `role="listitem"` inside a `role="list"` container
- Restriction checkboxes: native `<input type="checkbox">` with visible `<label>` (no custom reimplementation)
- Warning badge on SlotCard: `aria-label="[N] dietary conflicts"` on the badge button
