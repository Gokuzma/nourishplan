## Component Inventory

### New: ScheduleSection (Settings)

Slot in `SettingsPage.tsx` alongside `DietaryRestrictionsSection` and `WontEatSection`.

Props mirror the per-member pattern exactly:
```
householdId: string
memberId: string
memberType: 'user' | 'profile'
memberName?: string
```

Structure:
1. `<h4>` heading: `{memberName — }Weekly Schedule`
2. 7-column × 4-row grid (days across top, meal slots down left)
3. Day headers: Mon Tue Wed Thu Fri Sat Sun (3-letter abbreviations, `text-xs text-text/50`)
4. Row labels: Breakfast / Lunch / Dinner / Snack (`text-xs text-text/60`, left-aligned)
5. Each cell: tap-to-cycle through `prep → consume → quick → away → prep`
6. Cell renders as a square button with status-colored background (see Color section)
7. Status label inside cell: 1–4 character abbreviation (`Prep` / `Con` / `Quick` / `Away`), `text-[10px]` weight 600
8. Save button: `bg-primary text-white px-4 py-2 rounded-[--radius-btn] text-sm mt-3`
9. Exception toggle: inline date picker row below the grid — "Add exception for a specific date" link expands a date input + slot override form

### New: ScheduleBadge (SlotCard)

Inline with existing violation badge cluster (lines 92–96 of SlotCard.tsx).

Renders only when `scheduleStatus` is `consume`, `quick`, or `away` (not `prep`).

Dot size: `w-3 h-3 rounded-full` (12px, matching existing `w-4 h-4` badge scaled down for a dot-only indicator with no count).

Position: rendered immediately after the meal name, before the violation count badge.

New prop on `SlotCard`: `scheduleStatus?: 'prep' | 'consume' | 'quick' | 'away'`

### Modified: PlanGrid / DayCard

Passes `scheduleStatus` down to each `SlotCard` based on the selected member's schedule for that day + slot combination. Data flows through the same member-context path as suggestions and targets.

## Interaction Contract

### Grid picker (ScheduleSection)

- Tap/click cycles cell through statuses: `prep → consume → quick → away → prep`
- No drag — tap-only, one-tap per cycle step
- On mobile: cells are minimum 44px × 44px touch target
- Grid scrolls horizontally if viewport is narrower than 7 columns (mobile): `overflow-x-auto` on grid wrapper
- "Repeat last week's pattern" is out of scope for this phase (Phase 22 concern)
- Default for new members: all cells start as `prep`

### Exception overrides

- "Add exception" link below the recurring grid expands an inline form:
  - Date input (`<input type="date">`) styled `border border-secondary bg-background text-text text-sm px-3 py-2 rounded-[--radius-btn]`
  - A second mini-grid (same 7×4 layout, but single-week, pre-filled from recurring) for the selected week
  - "Save exception" and "Cancel" buttons
- Only one exception form open at a time
- Saved exceptions listed below as chips: `[Week of Apr 7] [×]` using the existing chip pattern from DietaryRestrictionsSection (line 95–106)

### Save behavior

- Schedule saves on "Save schedule" button press (no auto-save) — consistent with DietaryRestrictionsSection and WontEatSection save patterns
- Saving shows "Saving..." disabled state, then "Saved!" for 2 seconds — same pattern as SettingsPage display name save

### SlotCard badge

- Badge renders immediately when schedule data loads (no skeleton/delay)
- When `scheduleStatus` prop is absent or `prep`, no badge renders
- Badge has `title` attribute for tooltip: `consume` → "Pre-made from prep day", `quick` → "Quick meal only", `away` → "Away"

## Accessibility

- Each grid cell button: `aria-label="{Day} {Slot}: {current status}. Tap to change."` (e.g., "Monday Breakfast: Prep. Tap to change.")
- SlotCard badge `aria-label`: "Schedule: {status}" (e.g., "Schedule: away")
- Save button: `disabled` attribute during save, `aria-busy="true"`
- Grid wrapper: `role="grid"`, day header row: `role="row"`, each cell: `role="gridcell"`
