**Step 1: Push the database migration.**

Run: `SUPABASE_ACCESS_TOKEN=$SUPABASE_ACCESS_TOKEN npx supabase db push`

This creates the `plan_generations` table and adds `generation_rationale` column to `meal_plan_slots` in the live Supabase instance.

If the push command requires interactive confirmation for destructive changes, add `--include-all` flag. If it still prompts interactively, flag for manual intervention.

**Step 2: Deploy the generate-plan edge function.**

Run: `npx supabase functions deploy generate-plan --no-verify-jwt`

The `--no-verify-jwt` flag matches all other deployed functions (search-usda, search-cnf, analyze-ratings, classify-restrictions, verify-nutrition, delete-account). JWT verification is handled inside the function via `adminClient.auth.getUser()`.

**Step 3: Build and deploy the frontend.**

Run: `npx vite build && npx vercel --prod`

This deploys the updated PlanGrid, SlotCard, and all new components.

**Step 4: Verify deployment.**

- Confirm edge function is listed: `npx supabase functions list`
- Confirm migration was applied by checking the plan_generations table exists (use Supabase REST API or a quick test query)
