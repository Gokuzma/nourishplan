<objective>
Close UAT Gap A (test 1, truth: "generated plan fills all 28 weekly slots"). Currently every generation leaves 7 empty Snack slots because the UAT test household's 12 recipes are all full meals (Chicken Rice Bowl, Greek Salad, Tomato Soup, Vegetable Omelette, etc.), so the AI hallucinates recipe_ids for Snack slots and the validation filter drops them silently.

This plan makes three changes to `supabase/functions/generate-plan/index.ts`, layered on top of plan 22-06's timeout/status changes:

1. **Make the validation filter loud.** The current filter at lines 467-473 drops AI-returned assignments whose recipe_id doesn't match the household catalog, with zero traceability. Replace the silent filter with an explicit `droppedAssignments` array that captures `{day_index, slot_name, raw_recipe_id, reason}` for each drop. Write this array into `constraint_snapshot.droppedAssignments` so the UAT debugger can see exactly what the AI hallucinated.

2. **Add a slot-level fallback pass.** After Pass 2 validation (and after correction passes 3-5 if they ran), iterate over slotsToFill to find any slot that: (a) is not locked, (b) is not marked 'away' in the schedule, (c) has no valid assignment in bestResult.slots. For each such slot, attempt in order: (i) reuse a meal already assigned elsewhere in the plan (prefer one that was NOT assigned to the same slot_name on a nearby day), (ii) if no reuse candidate exists, push an entry into `suggestedRecipes` describing a canonical snack for that slot and mark the slot as `skippedForSuggestion` in `constraint_snapshot.skippedSlots`. This honors D-21 (suggest recipes to add) and D-22 (repeats allowed when catalog is small).

3. **Update the Pass 2 system prompt.** Add an explicit instruction that the AI must NEVER leave a Snack slot empty — instead either reuse a full-meal recipe OR surface a suggestedRecipe entry. This pushes the fallback into the AI's primary response and avoids most trips to the code-level fallback.

This plan does NOT add a schema-level `meal_type` column on recipes (the UAT "missing" list mentioned it as an option, but that's a larger phase of work — the code-level fallback covers the UAT gap). It does NOT touch PlanGrid or shimmer rendering (Gap D — plan 22-08). It does NOT touch test data or NutritionGapCard coverage (Gap C — plan 22-09).

      // Slot-level fallback (Gap A closure). For every slotToFill that ended up
      // without a valid assignment in bestResult.slots, attempt to reuse a meal
      // from elsewhere in bestResult.slots. If no reuse is possible, mark the
      // slot as skipped with a reason so the UI can show user-facing messaging.
      //
      // This honors D-22 (repeats allowed when catalog is small) and D-21
      // (suggest recipes the user could add).
      if (pass2Completed) {
        const assignedKeySet = new Set(
          bestResult.slots.map((s) => `${s.day_index}_${s.slot_name}`),
        );
        const awayKeySet = new Set(
          Object.entries(scheduleStatus)
            .filter(([, status]) => status === "away")
            .map(([k]) => k),
        );

        // Build a pool of candidate reuse assignments from bestResult.slots.
        // Prefer recipes that were NOT already assigned to the target slot_name
        // on any day, then fall back to any assigned recipe. This reduces visible
        // repetition of e.g. "Tomato Soup at Breakfast AND Snacks on same day".
        for (const slot of slotsToFill) {
          const key = `${slot.day_index}_${slot.slot_name}`;
          if (assignedKeySet.has(key)) continue;
          if (awayKeySet.has(key)) continue;

          // Try to find a reuse candidate
          const sameSlotNameUses = bestResult.slots.filter(
            (s) => s.slot_name === slot.slot_name,
          );
          const otherSlotNameUses = bestResult.slots.filter(
            (s) => s.slot_name !== slot.slot_name,
          );

          // Prefer a recipe not yet used in this slot_name (variety),
          // then any recipe from bestResult.slots.
          const reuseFrom =
            otherSlotNameUses.find((s) => !sameSlotNameUses.some((u) => u.recipe_id === s.recipe_id))
            ?? otherSlotNameUses[0]
            ?? sameSlotNameUses[0];

          if (reuseFrom) {
            bestResult.slots.push({
              day_index: slot.day_index,
              slot_name: slot.slot_name,
              recipe_id: reuseFrom.recipe_id,
              rationale: `Reused ${reuseFrom.rationale ? "(" + reuseFrom.rationale.slice(0, 60) + ")" : "from plan"} — catalog has no ${slot.slot_name.toLowerCase()}-specific recipe`,
            });
            reusedFills.push({
              day_index: slot.day_index,
              slot_name: slot.slot_name,
              meal_id: reuseFrom.recipe_id,
              source: "reuse_from_plan",
            });
            assignedKeySet.add(key);
          } else {
            // No reuse candidate — mark as skipped and ensure suggestedRecipes
            // has at least one generic snack entry so the user has an option.
            skippedSlots.push({
              day_index: slot.day_index,
              slot_name: slot.slot_name,
              reason: "no_catalog_recipe_and_no_reuse_candidate",
            });
          }
        }

        // If any Snacks slot was skipped AND suggestedRecipes doesn't already
        // contain at least one generic snack, seed a canonical snack suggestion.
        const snacksSkipped = skippedSlots.some((s) => s.slot_name === "Snacks");
        const hasSnackSuggestion = (suggestedRecipes ?? []).some((r) =>
          /snack|yogurt|trail mix|hummus|nut|fruit|granola/i.test(r.name + " " + r.description),
        );
        if (snacksSkipped && !hasSnackSuggestion) {
          suggestedRecipes = [
            ...(suggestedRecipes ?? []),
            {
              name: "Greek Yogurt with Berries and Granola",
              prepMinutes: 3,
              description: "Protein-rich snack: Greek yogurt topped with mixed berries and a tablespoon of granola. 180 kcal, 15g protein.",
            },
          ];
        }
