## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline pill spacing |
| sm | 8px | Compact element spacing, button icon padding (`p-2`) |
| md | 16px | Default element spacing, card padding (`px-4`) |
| lg | 24px | Section padding, card gaps (`py-6`) |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level bottom padding (tab bar clearance: `pb-[64px]`) |

Exceptions:
- Tab bar bottom clearance uses `pb-24` (96px) on PlanPage — match this value, not 64px
- Touch targets for icon buttons: minimum 28px (`w-7 h-7`) as used by existing `+` and `×` slot buttons
- Progress indicator step dots: 8px diameter with 4px gaps

Source: PlanPage.tsx `px-4 py-8 pb-24`, SlotCard.tsx `w-7 h-7`, CLAUDE.md page spacing convention.

## Component Inventory

New components required for this phase. All follow existing project patterns.

### GeneratePlanButton
- Placement: Plan page header row, immediately below week navigation, above MemberSelector
- Appearance: Full-width on mobile, fixed-width (180px) on desktop; `bg-primary text-white rounded-[--radius-btn] px-4 py-2.5 text-sm font-semibold`
- States: idle ("Generate Plan"), loading (disabled, shows spinner icon inline), completed (brief "Plan ready" flash then returns to idle)
- Disabled condition: when a generation job is in progress (`isPending`)

### PriorityOrderPanel
- Placement: Collapsible section directly below GeneratePlanButton, collapsed by default
- Toggle: chevron button labeled "Planning priorities" — same pattern as IssuesPanel
- Contents: drag-to-reorder list of 5 soft constraint labels: Nutrition, Preferences, Budget, Variety, Inventory
- Each row: drag handle (6-dot grid icon, `text-text/30`) + label (`text-sm text-text font-sans`) + position badge (`text-xs text-text/40`)
- Background: `bg-secondary rounded-[--radius-card] p-3`

### GenerationProgressBar
- Visibility: only during active generation job; hidden otherwise
- Placement: below GeneratePlanButton, above PriorityOrderPanel toggle
- Structure: thin 4px-height bar (`bg-primary/20` track, `bg-primary` fill) + step label below
- Step labels (shown beneath bar): "Shortlisting recipes...", "Assigning to slots...", "Verifying constraints...", "Finalising plan..."
- Step label style: `text-xs text-text/50 font-sans text-center`
- Animated: CSS `animate-pulse` on shimmer slots; bar fill transitions with `transition-all duration-500`

### SlotShimmer (generation skeleton state)
- Replaces each unlocked empty SlotCard during generation
- Appearance: matches SlotCard dimensions; `bg-secondary/60 animate-pulse rounded-lg` with a thin `bg-primary/20` left border strip (4px wide)
- Locked slots remain rendered normally during generation (not replaced with shimmer)

### AIRationaleTooltip
- Trigger: tap on a generated slot card (mobile) or hover (desktop)
- Appearance: small popover anchored below the slot card; `bg-surface border border-accent/30 rounded-[--radius-card] shadow-md px-3 py-2 text-xs text-text/70 font-sans max-w-[240px]`
- Content: 1–2 sentences of AI rationale text, e.g. "High protein, uses chicken from inventory, rated 4.5 by family."
- Dismiss: tap/click outside; auto-dismiss after 5 seconds on mobile
- Only visible on slots generated in the current or most recent generation run

### NutritionGapCard
- Placement: below PlanGrid, above IssuesPanel
- Visibility: only after a generation has completed for this plan week; hidden on weeks without generation data
- Structure: collapsible panel (collapsed by default); header shows member count with gaps, e.g. "2 members below nutrition target"
- Header: `bg-secondary rounded-[--radius-card] px-4 py-3 flex items-center justify-between`
- Header icon: same warning triangle SVG as IssuesPanel (`text-amber-500`)
- Per-member row: member avatar/name + nutrient name + gap percentage + "Swap suggestion" inline action
- Gap indicator: `text-amber-500 text-xs font-semibold` for values below 90% threshold
- Swap suggestion: `text-primary text-xs underline` tap target — "Swap [Day] [Slot] → [Recipe]" — tapping applies the swap via useAssignSlot

### RecipeSuggestionCard
- Visibility: only when household has fewer than 7 recipes; shown below NutritionGapCard
- Appearance: `bg-secondary/50 border border-primary/20 rounded-[--radius-card] px-4 py-3`
- Header: "Add more recipes to improve suggestions" in `text-sm font-semibold text-text`
- Body: 2–4 AI-suggested recipe tiles, each showing recipe name + estimated prep time + "Add to my recipes" button
- "Add to my recipes" button: `text-xs text-primary bg-primary/10 rounded-[--radius-btn] px-2 py-1 hover:bg-primary/20`

### GenerationJobBadge
- Placement: small inline badge on the plan header (beside week range)
- Content: "Generated [relative time]" e.g. "Generated 3 min ago"
- Style: `text-xs text-text/40 font-sans` — no background, no border; purely informational

## Interaction Contracts

### Generate Plan Flow

1. User taps "Generate Plan" button
2. Button disables immediately; shows inline spinner (16px SVG, `animate-spin`, `text-white`)
3. GenerationProgressBar appears with "Shortlisting recipes..." label
4. All unlocked empty slots replace their SlotCard with SlotShimmer
5. Unlocked occupied slots also replace with SlotShimmer (they will be overwritten per D-12)
6. Locked slots remain as-is with their lock indicator visible
7. Progress label advances through 4 steps as Edge Function polls report progress
8. On job completion: shimmer slots resolve to populated SlotCards with AIRationaleTooltip data attached
9. GenerationProgressBar fades out (`opacity-0 transition-opacity duration-300`)
10. Button returns to idle state
11. NutritionGapCard renders below PlanGrid if any member gaps detected
12. If household has <7 recipes: RecipeSuggestionCard renders below NutritionGapCard

### Timeout / Error State

- If generation exceeds 10s wall-clock budget: best-so-far slots populate; progress bar shows "Best plan found (time limit reached)" in `text-amber-500`
- If Edge Function returns error: button returns to idle; error toast appears at top of page ("Plan generation failed. Try again.") using `bg-red-50 border border-red-200 text-red-700 text-sm rounded-[--radius-card] px-4 py-3`
- Partial slot population on timeout is valid — some slots may remain empty

### Per-Slot "Suggest Alternative"

- Trigger: overflow menu on an AI-generated SlotCard (three-dot icon, same as existing plan actions)
- Label: "Suggest alternative" — appended to existing swap/clear/log actions
- On tap: slot shimmer state activates for that single slot only; brief generation for that slot
- On completion: slot updates with new meal + new AIRationaleTooltip

### Priority Reorder

- PriorityOrderPanel uses touch-drag reorder (same @dnd-kit/sortable already in project)
- Order persists in localStorage key `plan-priority-order-{householdId}` — no DB write needed
- On reorder: panel shows brief "Saved" confirmation text (`text-xs text-primary`) for 1.5s then fades

| Element | Copy |

| Generate button loading state | "Generating..." |
| Progress step 1 | "Shortlisting recipes..." |
| Progress step 2 | "Assigning to slots..." |

| Generation error toast | "Plan generation failed. Try again." |
| NutritionGapCard header (no gaps) | not shown |

| RecipeSuggestionCard header | "Add more recipes to improve suggestions" |
| RecipeSuggestionCard CTA | "Add to my recipes" |
| GenerationJobBadge | "Generated {relative time}" e.g. "Generated 3 min ago" |
| PriorityOrderPanel toggle label | "Planning priorities" |
| Priority saved confirmation | "Saved" |
| AIRationaleTooltip preamble | none — AI text starts immediately, no label prefix |
| Per-slot "Suggest alternative" label | "Suggest alternative" |
| Empty plan with no recipes (<7) suggestion header | "Add more recipes to improve suggestions" |
| Locked slot tooltip | "Locked — won't be replaced by generation" |

Destructive actions in this phase:
- "Generate Plan" on a week with existing unlocked meals: this overwrites all unlocked slots. No modal confirmation. Instead: the locked slot mechanism (Phase 19) is the user's tool to protect slots they want to keep. Rule: "Lock what you want to keep" is documented in the tooltip on each LockBadge — no pre-generation confirmation dialog.

Source: CONTEXT.md D-08 through D-13, D-21 decisions.

## Accessibility Contract

- GeneratePlanButton: `aria-busy="true"` when loading, `aria-label="Generate weekly meal plan"`
- GenerationProgressBar: `role="progressbar"` with `aria-valuetext` set to current step label
- SlotShimmer: `aria-hidden="true"` (decorative loading state)
- AIRationaleTooltip: `role="tooltip"` with `id` referenced by `aria-describedby` on the trigger slot card
- NutritionGapCard: `aria-expanded` on collapsible header button
- PriorityOrderPanel: `aria-expanded` on collapsible header button; drag items have `aria-label="{Priority Name}, position {N} of 5"`
- Swap suggestion links: `aria-label="Swap {Day} {Slot} to {Recipe Name} to close {Member} protein gap"`
