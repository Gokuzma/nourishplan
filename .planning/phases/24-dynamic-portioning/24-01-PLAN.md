## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser localStorage → component state | Any other browser code could write malformed data to `plan-recipe-mix-{householdId}` |
| component state → generate-plan request body | Mix values cross to edge function as untrusted input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-01 | Tampering | RecipeMixPanel localStorage read | mitigate | try/catch around JSON.parse; validate every field is `Number.isFinite(...)`; fall back to defaults on any failure (see `<behavior>` test 5) |
| T-24-02 | Tampering | getRecipeMix return value sent to edge function | mitigate | Server-side normalization is owned by Plan 24-02 (edge function clamps and divides by sum). Client still validates sum approx 100 before returning. |
| T-24-03 | Information Disclosure | localStorage key scoping | accept | Mix values are non-sensitive preference data. Key is scoped by `householdId` to prevent cross-tenant confusion in shared browsers (matches PriorityOrderPanel convention). |
| T-24-04 | Denial of Service | Malformed localStorage causing component crash | mitigate | All localStorage access wrapped in try/catch; malformed JSON returns defaults rather than throwing |
</threat_model>
