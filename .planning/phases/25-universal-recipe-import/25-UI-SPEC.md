## Component Inventory

### 1. ImportRecipeModal

A bottom-sheet modal (mobile) / centered dialog (sm+) matching the existing `AddInventoryItemModal` shell pattern exactly.

**Structure:**
```
fixed inset-0 z-50
  backdrop: bg-black/40
  panel: fixed inset-0 flex items-end sm:items-center justify-center
    div: bg-surface rounded-t-2xl sm:rounded-2xl w-full sm:max-w-md p-6
```

**States:**
- `idle`: Text input + primary CTA visible. Input placeholder: "Paste a URL or recipe text…"
- `loading`: Primary CTA replaced by spinner + "Importing…" label. Input disabled. No close-on-backdrop.
- `error`: Inline error message below input. CTA re-enabled. Error copy per Copywriting Contract below.
- `success`: Modal closes. Navigation to RecipeBuilder fires immediately.

**Input field:**
- `<textarea>` (not `<input type="text">`) — URLs are short but raw recipe text can be multiple paragraphs.
- Height: 4 rows minimum, auto-expand up to 8 rows.
- Styling matches existing inputs: `w-full rounded-[--radius-btn] border border-secondary bg-background px-3 py-2 text-sm text-text focus:outline-none focus:border-accent`

**Loading spinner:**
- Inline within the submit button: `opacity-70` text + a simple rotating border ring using Tailwind `animate-spin`.
- No full-screen overlay. User can still read the modal content during fetch.

**Backdrop click during loading:** Does NOT close modal (fetch in-flight).

---

### 2. RecipeBuilder Import Skeleton

Shown immediately after navigation to `/recipes/:id` following an import. Displayed while the edge function response populates RecipeBuilder fields.

**Pattern:** Matches existing inline skeleton in `RecipeBuilder.tsx` line 822 — `animate-pulse bg-secondary/50 rounded-[--radius-btn]`.

**Skeleton blocks to show (top to bottom):**
1. Recipe name field: `h-9 w-3/4 animate-pulse bg-secondary/50 rounded-[--radius-btn]`
2. Servings field: `h-9 w-1/4 animate-pulse bg-secondary/50 rounded-[--radius-btn]`
3. Ingredients section heading: `h-5 w-24 animate-pulse bg-secondary/50 rounded-[--radius-btn]`
4. Three ingredient rows: `h-14 animate-pulse bg-secondary/50 rounded-[--radius-btn]` × 3
5. Steps section heading: `h-5 w-16 animate-pulse bg-secondary/50 rounded-[--radius-btn]`
6. Two step rows: `h-12 animate-pulse bg-secondary/50 rounded-[--radius-btn]` × 2

Skeleton is replaced field-by-field as AI response streams in. Once all fields are populated, skeleton disappears and normal RecipeBuilder content is shown with no transition delay.

---

### 3. Source URL Attribution (Recipe Detail)

Displayed in RecipeBuilder / recipe detail page once `source_url` is present on the recipe record.

**Position:** Below the recipe title and notes, above the ingredients list.

**Markup:** A single line: `text-xs text-text/40` with a plain anchor `href={source_url}` target `_blank` rel `noopener noreferrer`.

**Copy:** "Imported from [domain]" — extract domain from the URL for display (e.g. `www.allrecipes.com`). If URL cannot be parsed, fall back to full URL truncated at 60 chars with ellipsis.

**Visibility:** Only when `source_url` is non-null. No badge on RecipesPage list cards (D-13).

### 4. RecipesPage Button Layout

Two buttons side by side in the header row:
- Existing "New Recipe" button (unchanged, right side)
- New "Import Recipe" button (left of "New Recipe", or stacked below on very small screens under 360px)

**Import Recipe button styling:** matches secondary/ghost variant — `rounded-[--radius-btn] border border-primary text-primary px-4 py-2 text-sm font-semibold hover:bg-primary/10 transition-colors` — visually distinct from the filled "New Recipe" primary button to indicate secondary action.

## Interaction Contracts

### Import Flow (happy path)

1. User taps "Import Recipe" on RecipesPage.
2. `ImportRecipeModal` opens — focus moves to textarea immediately (autofocus).
3. User pastes URL or text. No format validation before submit — accept any non-empty string.
4. User taps "Import Recipe" button.
5. Modal enters loading state: textarea disabled, button shows spinner + "Importing…".
6. Edge function returns recipe ID.
7. Modal closes. `navigate(/recipes/${recipeId})` fires.
8. RecipeBuilder renders with import skeleton active.
9. Fields populate as data arrives. Skeleton blocks replaced field by field.
10. RecipeBuilder is fully interactive once all fields are populated.

### Import Flow (error path)

1. Steps 1–5 identical to happy path.
2. Edge function returns an error OR fetch times out.
3. Modal stays open. Loading state ends. Error message appears below textarea.
4. User can edit the input and retry, or close the modal.

### Accessibility

- Modal: `role="dialog"` `aria-modal="true"` `aria-labelledby` pointing to the `<h2>` heading.
- Focus trap: on open, focus moves to textarea. On close, focus returns to the "Import Recipe" trigger button.
- Backdrop click when idle: closes modal (matches existing modal pattern).
- Escape key: closes modal when idle; no-op during loading.
- Skeleton blocks: `aria-hidden="true"` so screen readers do not announce them.
- Source attribution link: `aria-label="View original recipe source"` on the anchor.

| Element | Copy |
|---------|------|
| Modal heading | Import Recipe |

| Primary CTA (loading) | Importing… |
| CTA (cancel / secondary) | Discard Import |
| RecipesPage button label | Import Recipe |
| Success — no message shown | (silent — navigation is the feedback) |
| Error: URL fetch failure | Could not fetch that URL. Try copying and pasting the recipe text directly instead. |
| Error: AI parse failure | The import did not return a complete recipe. Try pasting the recipe text directly. |

| Empty state on RecipesPage (no recipes, not specific to import) | No recipes yet. Create your first one above. (unchanged) |
| Source attribution (recipe detail) | Imported from [domain] |
| RecipeBuilder skeleton aria label | Importing recipe, please wait… (on wrapper `aria-live="polite"`) |

No destructive actions in this phase. No delete confirmation copy required.
