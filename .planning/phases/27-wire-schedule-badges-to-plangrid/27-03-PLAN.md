      expect(within(monDayCard as HTMLElement).queryByLabelText('Schedule: away')).toBeNull()

    Why this selector is stable:
    - `rounded-[--radius-card]` is a Tailwind arbitrary-value class specific to DayCard's outer wrapper (see DayCard.tsx line 131). Inner elements use `rounded-lg`, `rounded-full`, etc., so `[class*="rounded-"]` alone is ambiguous.
    - `bg-surface` only appears on the DayCard root in this component — inner slot rows use `bg-background/50`, `bg-primary/10`, etc.
    - Compound selector `[class*="rounded-"][class*="bg-surface"]` uniquely matches the DayCard root.
    - Does NOT rely on adding `data-testid` or `data-day-index` attributes (avoids touching Plan 02 production code from Plan 03).

    If this selector fails to match after running the test (e.g. DOM structure changed), inspect via `screen.debug()` ONCE and adjust the selector — but do NOT weaken the assertion. The positive+negative pair is the whole point: it catches off-by-one shifts silently (if Plan 02 used `i` instead of `(weekStartDay + i) % 7`, with `weekStartDay=1` and row `day_of_week=2`, the dot would land on the Monday column (dayIndex=2 → Wed? no, `i=2` and plan-relative → displayed 'Wed') — the negative assertion fails, exposing the bug immediately).
