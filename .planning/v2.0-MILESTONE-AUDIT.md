## Cross-Phase Integration Findings

### CRIT-01 — Cook Mode from Plan does not trigger inventory deduction or spend log

**Affects:** INVT-05, INVT-06, BUDG-03
**Where:** `src/pages/CookModePage.tsx` (completion handler lines 175-178, 214-217)

The `handlePrimaryAction` in CookModePage reaches `useCompleteCookSession` which only sets `cook_sessions.status = 'completed'`. It does **not** invoke `useInventoryDeduct`, `useSpendLog`, or render `CookDeductionReceipt`. The only correctly-wired cook path is `RecipeBuilder.tsx:590-613` (the "Mark as Cooked" button on the recipe detail page).

**Consequence:** Any meal a user cooks via Plan → Cook button is invisible to:
- Inventory (no ingredient deduction, INVT-05)
- Budget (no spend_log entry, BUDG-03 weekly spend understates)
- Leftover flow (no "save leftovers?" prompt, INVT-06)

**Fix pattern:** Mirror RecipeBuilder's `handleMarkAsCooked` — call `spendLog.mutate(...)` then `inventoryDeduct.mutateAsync` in `onSuccess`, render `CookDeductionReceipt` before navigating back.

### CRIT-02 — PlanGrid never passes slotSchedules prop to DayCard

**Affects:** SCHED-01, SCHED-02 (UI feedback)
**Where:** `src/components/plan/PlanGrid.tsx` (dayCards render site, lines 524-557)

`DayCard.tsx` accepts `slotSchedules?: Map<string, ScheduleStatus>` (lines 74, 101, 171, 206) and correctly forwards `scheduleStatus` into each `SlotCard`. But `PlanGrid.tsx` never imports `useSchedule`, never builds the schedule map, and never passes the prop to DayCard at render time. Every call site receives `undefined`, so the schedule badges on slot cards are permanently blank.

Phase 21's VERIFICATION.md reported this wiring as VERIFIED (line 77 of 21-VERIFICATION.md reads "`PlanGrid.tsx` | `useSchedule.ts` | useSchedule hook call | ✓ WIRED"). This was incorrect at the time of verification — the code snippet that followed referenced `slotSchedulesByDay` but that variable is declared on an outdated version of PlanGrid.tsx; the current `main` branch PlanGrid does not contain it.

**Consequence:** SCHED-01 UI is inert. Users set schedules but the plan grid shows no reflection. SCHED-02 engine-side constraint works (generate-plan reads `member_schedule_slots`) but the UI feedback loop is dead.

**Fix:** In `PlanGrid.tsx`, import `useSchedule`, build `Map<string, ScheduleStatus>` keyed by `${day_index}:${slot_name}`, pass to each DayCard via the existing `slotSchedules` prop.

### WARN-01 — Orphaned edge functions (Phase 23)

`supabase/functions/generate-cook-sequence/index.ts` and `supabase/functions/generate-reheat-sequence/index.ts` have **zero** consumers in `src/`. CookModePage's multi-meal-prompt and reheat paths use local `liveSteps` from `useRecipeSteps` and a hardcoded 3-step fallback respectively. PREP-02's "day-of task sequencing" is only partially met for the combined-multi-recipe case.

Either wire both functions into CookModePage (combined multi-recipe step interleaving + AI-generated reheat sequence) or remove the orphans and update 23-SUMMARY.

Phase 25 shipped and UAT-approved, but the traceability table stops at `PORT-02`.
